### ① 核心痛点与破局点

- **痛点**：题目要求 `get` 和 `put` 必须在 **$O(1)$ 平均时间复杂度**内运行。
    
    - 如果只用**数组/链表**：找元素要 $O(n)$，无法做到 $O(1)$ 的 `get`。
        
    - 如果只用**哈希表（HashMap）**：虽然 `get` 是 $O(1)$，但哈希表是**无序**的，我们无法知道哪个数据是“最久未使用”的，无法在 $O(1)$ 内淘汰老数据。
        
- **破局点（数据结构合体）**：**哈希表 + 双向链表（叫“哈希链表”）**。
    
    - **双向链表**：用来维护数据的**时间顺序**。靠近头部的节点是最近使用的，靠近尾部的节点是最久未使用的。由于是双向的，删除和插入节点都是 $O(1)$。
        
    - **哈希表**：Key 是缓存的键，Value 是指向**链表节点的指针（iterator/pointer）**。有了它，我们就能瞬间定位到链表中的某一个节点，把 $O(n)$ 的链表查找优化成 $O(1)$。
        

### ② 整体思路：移花接木

我们将双向链表的头部定义为 **“最新使用”**，尾部定义为 **“最久未使用”**：

1. **`get(key)`**：
    
    - 如果哈希表中没有这个 key，直接返回 -1。
        
    - 如果有，通过哈希表瞬间找到链表节点，拿到 value。然后把这个节点**切断并移动到链表头部**（因为它刚刚被访问了，变成最新的了）。
        
2. **`put(key, value)`**：
    
    - 如果 key 已经存在：修改对应节点的值，并将该节点**移动到链表头部**。
        
    - 如果 key 不存在：
        
        - 如果缓存满了（达到 `capacity`）：把链表**尾部**的节点（最久未使用）删掉，同时在哈希表中也删掉它。
            
        - 创建一个新节点，插入到链表**头部**，并存入哈希表。
## 解题思路
1. 首先得重新定义数据结构Node，这是双向链表 + 哈希表的结合体。然后需要四个辅助函数，`moveToHead(Node* node)` `addToHead(Node* node)` `removeNode(Node* node)` `removeTail()`。
2. 完成初始化，get和put方法。


### ③ 完整 C++ 代码（双向链表 + 哈希表）

面试时，为了避免复杂的指针断开和拼接出错，强烈建议引入 **`dummy_head`（伪头）** 和 **`dummy_tail`（伪尾）**。

C++

```
#include <unordered_map>

class LRUCache {
private:
    // 定义双向链表节点
    struct Node {
        int key, value;
        Node* prev;
        Node* next;
        Node() : key(0), value(0), prev(nullptr), next(nullptr) {}
        Node(int k, int v) : key(k), value(v), prev(nullptr), next(nullptr) {}
    };

    int cap;
    Node* head; // 伪头
    Node* tail; // 伪尾
    std::unordered_map<int, Node*> cache; // 哈希表：key -> 节点指针

    // 辅助函数 1：把一个已知节点移动到头部（先断开，再插到头）
    void moveToHead(Node* node) {
        removeNode(node);
        addToHead(node);
    }

    // 辅助函数 2：在头部插入节点
    void addToHead(Node* node) {
        node->prev = head;
        node->next = head->next;
        head->next->prev = node;
        head->next = node;
    }

    // 辅助函数 3：删除一个已知节点
    void removeNode(Node* node) {
        node->prev->next = node->next;
        node->next->prev = node->prev;
    }

    // 辅助函数 4：淘汰尾部节点
    Node* removeTail() {
        Node* res = tail->prev; // 真正的尾节点
        removeNode(res);
        return res;
    }

public:
    LRUCache(int capacity) : cap(capacity) {
        head = new Node();
        tail = new Node();
        head->next = tail;
        tail->prev = head;
    }
    
    int get(int key) {
        if (!cache.count(key)) return -1;
        Node* node = cache[key];
        moveToHead(node); // 访问了就要刷新位置
        return node->value;
    }
    
    void put(int key, int value) {
        if (cache.count(key)) {
            // key 存在，改值，调到头
            Node* node = cache[key];
            node->value = value;
            moveToHead(node);
        } else {
            // key 不存在，建新节点
            Node* newNode = new Node(key, value);
            cache[key] = newNode;
            addToHead(newNode);
            // 超容则淘汰
            if (cache.size() > cap) {
                Node* removed = removeTail();
                cache.erase(removed->key); // 必须在哈希表中也删掉
                delete removed; // 释放内存
            }
        }
    }
};
```

### ④ 关键细节与面试追问

- **Q：为什么链表节点里必须同时存 `key` 和 `value`？只存 `value` 行不行？**
    
    - **A**：**不行！** 当缓存满了需要淘汰尾部节点时，我们通过 `tail->prev` 找到了链表节点。但我们同时需要**去哈希表里把这个键值对删掉**。如果节点里不存 `key`，哈希表就找不到该删谁。
        
- **Q：C++ 标准库里有现成的双向链表 `std::list`，面试可以偷懒用吗？**
    
    - **A**：面试官一般要求手写双向链表。因为手写链表能展现你极强的指针操作基本功。如果时间不够，可以先和面试官沟通：“我可以用 `std::list` 配合 `std::unordered_map` 快速实现吗？” 如果同意，可以用 `std::list<pair<int, int>>` 来写。
        

### ⑤ 避坑指南（面试挂点）

1. **删除尾节点时忘记同步删哈希表**：这是最容易漏掉的一步，导致哈希表越撑越大。
    
2. **指针操作顺序写错**：在 `addToHead` 时，必须先修改新节点的 `prev` 和 `next`，再改原有节点的指针，否则容易发生指针丢失或死循环。
