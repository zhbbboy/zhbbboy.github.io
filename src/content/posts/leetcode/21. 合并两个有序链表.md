### ① 核心思路：双指针 + 比较取小

两条链表都是升序的，所以每次比较两个链表当前节点，**把较小的那个接到结果链表后面**，然后该指针前进一步。一直重复，直到某条链表走完，再把另一条剩余部分整体接上。

---

### ② 关键技巧：虚拟头节点 dummy

**痛点：** 如果不用 dummy，你需要先判断哪条链表的头更小来确定结果链表的头，代码会有很多 if 特判，很啰嗦。

**解决：** 创建一个虚拟头节点 `dummy`，它不存有效数据，只是一个"占位的起点"。所有节点都接在它后面，最后返回 `dummy->next` 就是真正的头。这样**省去了对头节点的特殊处理**。

## 解题思路
#### 一.迭代法
1. 首先最终要的是得找一个虚拟头节点dummy当res链表的头节点，如果不用 dummy，你需要先判断哪条链表的头更小来确定结果链表的头，代码会有很多 if 特判，很啰嗦。所以就先创建一个带虚拟头节点的链表，然后再创建一个指针指向这个链表的尾部，始终指向结果链表的最后一个节点。
2. 进入比较大小插入节点的循环，循环的条件就是必须两个链表遍历的节点都不为空，否则就代表着有个链表已经被遍历完了并且，后续就直接将另外个链表直接插入结果链表的尾部就ok了。
3. 最后返回的是dummy.next。


### ③ 迭代法完整代码

cpp

```cpp
ListNode* mergeTwoLists(ListNode* list1, ListNode* list2) {
    // 虚拟头节点，避免处理头节点的特殊情况
    ListNode dummy(0);
    ListNode* tail = &dummy;  // tail 始终指向结果链表的最后一个节点
    
    // 两条链表都还有节点时，比较取小
    while (list1 != nullptr && list2 != nullptr) {
        if (list1->val <= list2->val) {
            tail->next = list1;     // 把 list1 节点接上
            list1 = list1->next;    // list1 前进
        } else {
            tail->next = list2;
            list2 = list2->next;
        }
        tail = tail->next;          // tail 前进到新接上的节点
    }
    
    // 其中一条走完了，把另一条剩余的整体接上
    tail->next = (list1 != nullptr) ? list1 : list2;
    
    return dummy.next;  // 真正的头节点
}
```

---

### ④ 几个关键细节

**`tail->next = (list1 != nullptr) ? list1 : list2;` 为什么能直接整体接上？**

因为两条链表本身都是有序的。当一条走完，另一条剩下的部分一定都比已经接入的所有节点大（或相等），而且它自己内部有序，所以可以直接整段拼接，不用再逐个比较。

**为什么用 `ListNode dummy(0)` 而不是 `new ListNode(0)`？**

`ListNode dummy(0)` 是栈上对象，函数结束自动销毁，无需手动 delete，更安全。用 `new` 会在堆上分配，需要记得释放（但因为返回的是 `dummy.next`，dummy 本身可以丢弃）。栈对象写法更简洁。

**`<=` 还是 `<`？相等时取哪个？**

用 `<=`，相等时取 list1。这能保证**稳定性**（相等元素保持原有相对顺序），虽然这道题不强制要求，但是好习惯。

### 二.递归法
1. 终止条件，就是任意个链表为空，返回另外一条链表。
2. 递归调用，当前层的处理：
- 比较两个头节点，假设 `list1->val` 更小
- 那么合并后的链表，头节点一定是 `list1`（因为它最小）
- 接下来 `list1` 后面应该接什么？应该接"`list1->next` 和 `list2` 合并后的结果"
- 这正是 `mergeTwoLists(list1->next, list2)`——**我信任它能返回正确的合并结果**
- 把这个结果接到 `list1->next`，然后返回 `list1`

### ⑤ 递归法（更简洁）


```cpp
ListNode* mergeTwoLists(ListNode* list1, ListNode* list2) {
    // 终止条件：任一链表为空，返回另一条
    if (list1 == nullptr) return list2;
    if (list2 == nullptr) return list1;
    
    // 取较小的头节点，它的 next 指向"剩余部分合并的结果"
    if (list1->val <= list2->val) {
        list1->next = mergeTwoLists(list1->next, list2);
        return list1;
    } else {
        list2->next = mergeTwoLists(list1, list2->next);
        return list2;
    }
}
```

递归思路：每次选出两个头中较小的，让它的 next 连接"剩下的合并结果"。代码极简，但空间 O(n)（递归栈），迭代是 O(1)。

### ⑥ 面试常见追问

**Q：时空复杂度？**

时间 O(m+n)，每个节点访问一次。迭代法空间 O(1)，递归法空间 O(m+n)（栈深度）。

**Q：合并 K 条有序链表怎么办（23 题）？**

可以两两合并（用本题作为子函数），或者用优先队列（最小堆）每次取 K 个头中最小的。这道题是 23 题的基础。

**Q：这个题和归并排序什么关系？**

归并排序的"合并"步骤就是这个操作。148 题"排序链表"就是用归并排序，其中合并部分直接复用本题代码。
