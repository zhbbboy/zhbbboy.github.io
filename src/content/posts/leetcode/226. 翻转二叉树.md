### ① 核心痛点与破局点

- 🛑 **痛点**：二叉树的翻转是“牵一发而动全身”的。如果只交换了根节点的左右两个孩子，它们底下的子孙节点如果没有跟着翻转，整棵树的相对位置就会错乱。
    
- 🔄 **破局点（宏观思维）**：不要陷入细节。相信递归的力量！
    
    - 站在当前节点视角：我只需要把我的**左指针**和**右指针**互换。
        
    - 至于左子树内部怎么翻转、右子树内部怎么翻转，直接交给**递归函数**去处理。
        

### ② 整体思路：分治与递归

我们可以将大任务分解为相同的子任务：

1. 🛑 **基线条件（Base Case）**：如果当前节点是 `nullptr`（空树），不需要翻转，直接返回 `nullptr`。
    
2. 🔄 **递归步骤（Recursive Step）**：
    
    - 递归翻转左子树，拿到翻转后的新左子树。
        
    - 递归翻转右子树，拿到翻转后的新右子树。
        
    - **核心操作**：将当前节点的 `left` 指向新右子树，将 `right` 指向新左子树（即两两交换）。
        
    - 返回当前节点 `root`。
## 解题思路
1. 终止条件：如果遍历到的节点是空节点，那就不翻转直接返回nullptr。
2. 递归步骤：将左子树和右子树都送去翻转，并拿到翻转后的结果。
3. 核心操作：将root的左子树指针指向翻转好了的右子树。将右子树指针指向翻转好了的左子树。

### ③ 完整 C++ 代码（经典递归版）

```
class Solution {
public:
    TreeNode* invertTree(TreeNode* root) {
        // 要素 1：基线条件（出口）
        if (root == nullptr) {
            return nullptr;
        }

        // 要素 2：递归步骤（推进）
        // 先把左右子树分别送去翻转，并拿到翻转后的结果
        TreeNode* leftInverted = invertTree(root->left);
        TreeNode* rightInverted = invertTree(root->right);

        // 核心操作：交换当前节点的左右子树指针
        root->left = rightInverted;
        root->right = leftInverted;

        // 返回搞定后的自己
        return root;
    }
};
```

### ④ 关键细节与面试追问

- **Q：交换指针的时候，可以不定义中间变量 `leftInverted` 吗？**
    
    - **A**：如果直接写 `root->left = invertTree(root->right); root->right = invertTree(root->left);` 会**出错**！因为第一行执行完后，`root->left` 已经被修改了，第二行再去递归 `root->left` 时，实际上递归的是原本的右子树。如果要在一行内写完，在 C++ 中可以使用 `std::swap(root->left, root->right)`，然后再分别递归它们。
