## 解题思路
1. 首先定义边界上边界top=0，下边界bottom=m-1，左边界left=0，右边界right=n-1.
2. 定义大循环，因为螺旋矩阵是遍历次数可能有多圈的。循环的条件就是得上边界小于等于下边界并且左边界小于等于右边界，如果二者不同时满足则退出循环
3. 向右遍历，遍历的坐标是列坐标j，从左向右遍历的话，那初始值j就是left。当遍历到最右边的时候退出循环，所以循环条件就是j<=right。执行的动作就是依次```result.push_back(matrix([top][j])```,最后得top++，因为top行已经遍历完了
4. 向下遍历，遍历的坐标是行坐标i，从上向下遍历的话，那初始值就是top了。当遍历到最下面的时候退出循环，所以循环条件是i<=bottom,这里解释一下为什么不需要像向左一样先if判断一下top<=bottom，那是因为for循环里面本身就有top<=bottom的循环。执行的动作就是依次```result.push_back(matrix[i][right]);```最后需要right--，因为right列已经遍历完了。
5. 向左遍历，因为前面有top++了并且for循环里面没有判断top与bottom的语句，所以开头得先if判断，虽然right--了，但是for循环里面有相关的条件判断所以就不需要额外去判断了。遍历的坐标是列坐标j，从右向左遍历的话，那初始值j就是right。当遍历到最左边的时候退出循环，所以循环条件就是j>=left。执行的动作就是依次```result.push_back(matrix[bottom][j]);```,最后得bottom--，因为bottom行已经遍历完了
6. 向右遍历同上思路



### ③ 最容易出错的地方：步骤③④为什么要加判断

这是这道题的核心难点，用例子说清楚：

**场景：只剩一行**（如 `top==bottom`）

```
遍历完 ① 向右后，top++，此时 top > bottom
如果不判断直接执行 ③ 向左，就会把这一行重复遍历一遍！
```

**场景：只剩一列**（如 `left==right`）

```
遍历完 ② 向下后，right--，此时 right < left
如果不判断直接执行 ④ 向上，就会把这一列重复遍历一遍！
```

**记忆方法：**

- ①② 不需要判断，因为 while 条件保证了进入循环时一定有行列剩余
- ③ 执行前 top 已经 +1，所以需要重新检查 `top <= bottom`
- ④ 执行前 right 已经 -1，所以需要重新检查 `left <= right`

### ④ 面试常见追问 + 答案

**Q：这道题有没有更简洁的写法？**

有，方向数组法。用 `dx[]={0,1,0,-1}` 和 `dy[]={1,0,-1,0}` 表示右/下/左/上，碰壁就换方向。代码行数更少但不如边界法直观，面试中边界法更容易讲清楚。

**Q：时间和空间复杂度？**

时间 O(mn)，每个元素访问一次。空间 O(1)（不算输出数组），只用了四个边界变量。

**Q：如果是 59 题"螺旋矩阵 II"（往矩阵里填数字而不是读出来）怎么办？**

完全一样的边界逻辑，只是把 `result.push_back(matrix[i][j])` 改成 `matrix[i][j] = num++` 即可，框架不变。
