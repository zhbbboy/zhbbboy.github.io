### ① 破题：朴素思路 vs 一趟扫描

**朴素思路（两趟）：** 先遍历一遍数出链表总长度 `L`，倒数第 n 个就是正数第 `L-n+1` 个，再遍历一遍删除。能做，但要两趟。

**一趟扫描思路：** 让两个指针拉开 `n` 的固定间隔，当快指针到达末尾时，慢指针正好停在要删除节点的**前一个**位置。

---

### ② 核心技巧：让两指针保持 n 的间隔

关键是想清楚一个数学关系：

```
让 fast 先走 n+1 步，然后 fast 和 slow 一起走
当 fast 走到 null（末尾的下一个），slow 正好在"待删节点的前一个"
```

为什么是前一个？因为删除一个节点，需要拿到它的**前驱节点**，才能执行 `prev->next = prev->next->next` 跳过它。
![](../_Pasted_Image/Pasted%20image%2020260625121331.png)
## 解题思路
1. 首先得为该链表创建一个头节点，其next指针指向head，是为了方便删除第一个节点的
2. 然后再定义fast和slow指针分别指向dummy头节点
3. 再将fast指针走向与与slow指针相隔n+1的位置，目的是为了当fast指针走向nullptr的时候，slow指针走向了倒数第n个节点的前一个节点
4. while循环遍历，fast和slow指针都前进，当fast走到nullptr的时候，slow也刚好走到了倒数第n个节点的前一个节点
5. slow->next = slow->next->next;去删除倒数第n个节点
6. 返回dummy.next也就是删除倒数第n个节点后的链表

### ③ 完整 C++ 代码

cpp

```cpp
ListNode* removeNthFromEnd(ListNode* head, int n) {
    // 虚拟头节点，处理"删除头节点"的边界情况
    ListNode dummy(0);
    dummy.next = head;
    
    ListNode* fast = &dummy;
    ListNode* slow = &dummy;
    
    // fast 先走 n+1 步
    for (int i = 0; i < n + 1; i++) {
        fast = fast->next;
    }
    
    // 两指针一起走，直到 fast 到达 null
    while (fast != nullptr) {
        fast = fast->next;
        slow = slow->next;
    }
    
    // 此时 slow 是待删节点的前一个，执行删除
    slow->next = slow->next->next;
    
    return dummy.next;
}
```

一趟扫描，O(L) 时间，O(1) 空间。

### ④ 两个关键设计

**为什么 fast 要先走 `n+1` 步，而不是 `n` 步？**

我们要让 slow 停在**待删节点的前一个**（前驱），而不是待删节点本身。如果只走 n 步，slow 会停在待删节点上，但删除链表节点需要拿到它的前驱才能跳过它。多走一步（n+1），就让 slow 落后到前驱位置。

可以这样验证间隔：fast 和 slow 之间相差 n+1，当 fast 走到 null（即第 L+1 个虚拟位置），slow 在第 `L+1-(n+1) = L-n` 个位置，正好是倒数第 n 个的前一个。

**为什么必须用 dummy？**

考虑删除头节点的情况（示例3 `[1,2]` 删倒数第2个，即删除头节点 `1`）。如果没有 dummy，删头节点时找不到"头的前驱"，需要特殊处理。加上 dummy，头节点也有了前驱（就是 dummy），删除逻辑统一，无需特判。

### ⑤ 面试常见追问

**Q：题目保证 `1 <= n <= sz`，如果 n 可能越界怎么办？**

需要先判断 fast 走 n+1 步时是否提前为 null（说明 n 超过了链表长度），加一个边界检查。本题有约束所以不用，但工程中要考虑。

**Q：两趟扫描和一趟扫描哪个更好？**

时间复杂度都是 O(L)，但一趟扫描常数更小，且面试进阶要求就是一趟。两趟思路更直观，可以作为兜底答案先讲，再优化到一趟。

**Q：快慢指针在这道题和环形链表（141）里有什么区别？**

环形链表是"速度差"（一个走1步一个走2步，靠速度不同相遇）。这道题是"间隔差"（两个都走1步，但保持固定的位置间隔）。都叫双指针，但用法不同。
