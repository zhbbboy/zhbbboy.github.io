### ① 迭代法：三个指针的舞蹈

**核心思路：** 遍历链表，把每个节点的 `next` 指针方向反过来。需要三个指针：

```
prev  → 已经反转好的部分（初始为 nullptr）
curr  → 当前正在处理的节点
next  → 临时保存 curr 原本的下一个节点（防止断链后丢失）
```

每一步做三件事：**先存、再翻、再前进。**

### ② 迭代法完整代码

cpp

```cpp
ListNode* reverseList(ListNode* head) {
    ListNode* prev = nullptr;
    ListNode* curr = head;
    
    while (curr != nullptr) {
        ListNode* next = curr->next;  // ① 先存住下一个节点，否则断链后找不到了
        curr->next = prev;            // ② 反转指向：当前节点指向前一个
        prev = curr;                  // ③ prev 前进一步
        curr = next;                  // ④ curr 前进一步
    }
    
    return prev;  // curr 为 null 时，prev 正好停在新的头节点上
}
```

**记忆口诀：存、翻、进、进。** 四步顺序绝对不能乱，尤其是第①步必须在第②步之前，否则 `curr->next` 被覆盖后就永久丢失了原来的下一个节点。

### ③ 递归法：从尾部开始反转

**核心思路：** 假设后面的链表已经反转好了，只需要处理当前节点和它的下一个节点之间的关系。

```
reverseList(head) 的语义：反转以 head 为起点的链表，返回反转后的新头节点
```

递归三要素：

```
终止条件：head 为空，或只有一个节点 → 直接返回 head
递归调用：先反转 head 之后的部分，得到 newHead
当前处理：把 head 接到反转后链表的尾部
```

cpp

```cpp
ListNode* reverseList(ListNode* head) {
    // 终止条件：空链表或只有一个节点，不需要反转
    if (head == nullptr || head->next == nullptr) {
        return head;
    }
    
    // 先递归反转 head 后面的部分，newHead 是反转后的新头
    ListNode* newHead = reverseList(head->next);
    
    // head->next 这个节点（原本的下一个）现在变成了反转后链表的尾节点
    // 让它指向 head，完成 head 这一节的反转
    head->next->next = head;
    head->next = nullptr;  // head 现在是新的尾节点，断开旧的 next
    
    return newHead;
}
```

---

### ④ 递归过程图解

以 `[1,2,3]` 为例，理解递归是怎么"从尾部往回连"的：

```
reverseList([1,2,3])
  ↓ 递归调用 reverseList([2,3])
      ↓ 递归调用 reverseList([3])
          → 到达终止条件，直接返回 3（newHead=3）
      ← 回到 [2,3] 层：head=2, head->next=3
        执行：3->next = 2  （3 指向 2）
        执行：2->next = null
        返回 newHead = 3   （链表变成 3→2）
  ← 回到 [1,2,3] 层：head=1, head->next=2
    执行：2->next = 1  （2 指向 1）
    执行：1->next = null
    返回 newHead = 3   （链表变成 3→2→1）
```

**关键理解：** 递归是先一路递归到底（到达最后一个节点），再在"返回"的过程中，从后往前把每一节的指向反过来接上。

### ⑤ 面试常见追问 + 答案

**Q：迭代法和递归法的空间复杂度分别是多少？**

迭代法 O(1)，只用了三个指针变量。递归法 O(n)，因为递归调用栈深度等于链表长度，每层栈帧占用空间。

**Q：链表长度很长时（比如10万节点），用哪种方法？**

迭代法。递归法栈深度太大会导致栈溢出（Stack Overflow），这是递归处理链表问题的通病。

**Q：`head->next->next = head` 这一行最容易看晕，怎么理解？**

`head->next` 是原链表里 head 的下一个节点（比如 head=1 时，`head->next` 是节点2）。这行代码的意思是：让节点2指向节点1，也就是把"2→1"这条反转的边建立起来。
