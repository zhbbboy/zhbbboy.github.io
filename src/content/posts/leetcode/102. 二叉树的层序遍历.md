## ① 破题：什么是层序遍历？

题目给你一棵二叉树：

```
        3
       / \
      9   20
          / \
         15  7
```

要求返回：

```
[
  [3],
  [9, 20],
  [15, 7]
]
```

也就是：

```
第一层：3
第二层：9, 20
第三层：15, 7
```

所以这题不是普通的前序、中序、后序遍历，而是：

```
一层一层地访问节点
```

这种遍历方式叫：

```
BFS，广度优先搜索
```

二叉树层序遍历最常用的数据结构是：

```
queue<TreeNode*> q;
```

---

## ② 核心技巧：队列 + 当前层节点数量 size

这题最核心的是：

```
队列 queue 负责从左到右访问节点；
size 负责区分当前层和下一层。
```

为什么需要 `size`？

因为你处理当前层节点的时候，会把下一层节点放进队列。

比如当前队列是：

```
[9, 20]
```

这表示当前层有两个节点。

当你处理 `20` 的时候，会把它的孩子 `15` 和 `7` 放进队列。

如果你不提前记录当前层的节点个数，就容易把下一层节点也一起处理了。

所以每一轮 while 开始时，要先记录：

```
int size = q.size();
```

这个 `size` 表示：

```
当前这一层有多少个节点。
```

然后只循环 `size` 次。
## 解题思路
1. 首先创建一个二维数组res用来存储最终结果的。然后再处理特殊情况``root==nullptr``的时候，如果为空直接返回res就行。
2. 处理层序遍历的问题是需要用的队列的，所以先创建一个队列q，然后如果root不为空我们就可以先将root push到队列q中。
3. 开始层序遍历循环，循环退出条件肯定就是队列是否为空。首先由于层序遍历的结果是需要一层一层的保存的，所以我需要统计每层的数量，因为每次处理一个节点就要传入它们的孩子节点，所以不能根据队列来判断每层节点的数量。还要定义一个数组来存储每层节点的val。
4. 然后需要遍历每层的节点，所以要用for循环来遍历，用size来控制循环次数，然后每遍历一个节点时想将其保存到一个新节点中，然后再pop该队首。然后再将节点的val存入到level数组中，然后再将其左孩子和右孩子push到队列中如果有的话。然后遍历完该层后，再将level数组push到res二维数组中去。
5. 后续如此循环层序遍历完每层每个节点，最后return res二维数组。

# ④ 完整 C++ 代码

```cpp
class Solution {
public:
    vector<vector<int>> levelOrder(TreeNode* root) {
        vector<vector<int>> res;

        if (root == nullptr) {
            return res;
        }

        queue<TreeNode*> q;
        q.push(root);

        while (!q.empty()) {
            int size = q.size();
            vector<int> level;

            for (int i = 0; i < size; i++) {
                TreeNode* node = q.front();
                q.pop();

                level.push_back(node->val);

                if (node->left != nullptr) {
                    q.push(node->left);
                }

                if (node->right != nullptr) {
                    q.push(node->right);
                }
            }

            res.push_back(level);
        }

        return res;
    }
};
```

# ⑤ 关键设计解释

## 1. 为什么要判断 root == nullptr？

如果树为空：

```
root = nullptr
```

那就没有任何节点，直接返回空数组：

```
[]
```

所以：

```
if (root == nullptr) {
    return res;
}
```

---

## 2. 为什么用 queue？

队列是先进先出。

先进入队列的节点会先被处理。

这正好符合层序遍历：

```
先处理上一层，再处理下一层
同一层从左到右处理
```

---

## 3. 为什么要先存 size？

这是这题最重要的点。

```
int size = q.size();
```

表示当前层节点数量。

然后：

```
for (int i = 0; i < size; i++)
```

只处理当前层的节点。

在处理当前层时，新加入队列的孩子节点属于下一层，不能在本轮处理。

---

## 4. 为什么先 push 左孩子，再 push 右孩子？

因为题目要求每层从左到右。

所以代码顺序是：

```
if (node->left) q.push(node->left);
if (node->right) q.push(node->right);
```

如果你反过来写，结果顺序就可能变成从右到左。

# ⑥ 面试常见追问

## Q1：为什么不能直接 while 队列不空一直弹？

可以弹，但如果你不记录 `size`，你只能得到一个一维遍历结果：

```
[3, 9, 20, 15, 7]
```

但题目要的是按层分组：

```
[[3], [9, 20], [15, 7]]
```

所以必须用 `size` 区分每一层。

---

## Q2：为什么不能在 for 循环里写 `i < q.size()`？

不要这样写：

```
for (int i = 0; i < q.size(); i++)
```

因为 `q.size()` 会变化。

你在处理当前层节点时，会把下一层节点加入队列。

所以队列大小会不断改变。

正确做法是先固定当前层大小：

```
int size = q.size();

for (int i = 0; i < size; i++) {
    ...
}
```

---

## Q3：时间复杂度是多少？

每个节点只会进队一次、出队一次。

所以时间复杂度：

```
O(n)
```

其中 `n` 是树的节点数量。

空间复杂度：

```
O(n)
```

最坏情况下，队列里可能存放一整层的节点。

# ⑦ 复习版总结

你可以这样记：

```
102. 二叉树的层序遍历

解题思路：BFS + 队列。

1. 如果 root 为空，直接返回空数组。

2. 使用 queue<TreeNode*> 存储待访问节点，先把 root 放入队列。

3. 每一轮 while 处理一整层。
   在每轮开始时，用 size = q.size() 记录当前层节点数量。

4. 用 for 循环处理 size 个节点：
   - 取出队头节点；
   - 将节点值加入当前层 level；
   - 如果有左孩子，加入队列；
   - 如果有右孩子，加入队列。

5. 当前层处理完后，把 level 加入 res。

6. 队列为空时，说明整棵树遍历完成，返回 res。
```

核心记忆：

```
队列负责 BFS 顺序；
size 负责分层；
level 保存当前层；
res 保存所有层。
```
