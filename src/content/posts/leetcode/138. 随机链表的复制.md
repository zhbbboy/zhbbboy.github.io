这道题的难点不在链表遍历，而在 **random 指针的处理**——它可以指向任意节点，包括还没创建的节点。我们讲两种主流解法。

---

### ① 破题：难点在哪

普通链表深拷贝很简单，遍历一遍逐个 new 新节点就行。但这道题每个节点多了个 `random` 指针，指向链表中**任意位置**。

矛盾点：当你正在复制节点 A，A 的 random 指向节点 D，但 D 可能还没被创建。**你没法在创建 A 的时候就把 A 的 random 接好。**

解决思路就是要建立"**原节点 → 新节点**"的映射关系，这样不管 random 指向谁，都能通过映射快速找到对应的新节点。
## 解题思路
### ② 方法一：哈希表（最直观，推荐先掌握）

用一个哈希表 `map`，键是原节点，值是对应的新节点。

**分两趟：**

- 第一趟：遍历原链表，为每个原节点创建对应的新节点，存入哈希表（先不管指针）
- 第二趟：再遍历一遍，借助哈希表把每个新节点的 `next` 和 `random` 都接好

### ③ 哈希表法完整代码

```cpp
Node* copyRandomList(Node* head) {
    if (head == nullptr) return nullptr;
    
    unordered_map<Node*, Node*> map;  // 原节点 → 新节点
    
    // 第一趟：创建所有新节点，建立映射（先不接指针）
    Node* cur = head;
    while (cur != nullptr) {
        map[cur] = new Node(cur->val);
        cur = cur->next;
    }
    
    // 第二趟：接 next 和 random
    cur = head;
    while (cur != nullptr) {
        map[cur]->next = map[cur->next];      // cur->next 为 null 时 map[null]=null，安全
        map[cur]->random = map[cur->random];  // 同理
        cur = cur->next;
    }
    
    return map[head];  // 新链表的头
}
```

**为什么 `map[cur->next]` 不会出错？** 当 `cur->next` 是 nullptr 时，`unordered_map` 对不存在的键会返回默认值（指针类型默认是 nullptr），正好对应"新节点的 next 也是 null"，逻辑自洽。

时间 O(n)，空间 O(n)（哈希表）。

---

### ④ 方法二：节点交织（O(1) 空间，进阶最优解）

不用哈希表，而是把新节点**穿插**在原节点后面，利用位置关系建立映射。

**三步走：**

- 第一步：在每个原节点后面插入它的复制节点，`A→A'→B→B'→C→C'`
- 第二步：设置 random，新节点的 random 就是原节点 random 的下一个：`A'->random = A->random->next`
- 第三步：把交织链表拆分成原链表和新链表
```cpp
Node* copyRandomList(Node* head) {
    if (head == nullptr) return nullptr;
    
    // 第一步：在每个原节点后插入复制节点
    Node* cur = head;
    while (cur != nullptr) {
        Node* copy = new Node(cur->val);
        copy->next = cur->next;
        cur->next = copy;
        cur = copy->next;  // 跳到下一个原节点
    }
    
    // 第二步：设置复制节点的 random
    cur = head;
    while (cur != nullptr) {
        if (cur->random != nullptr)
            cur->next->random = cur->random->next;  // 复制节点的random = 原random的下一个
        cur = cur->next->next;
    }
    
    // 第三步：拆分两条链表
    Node* newHead = head->next;
    cur = head;
    while (cur != nullptr) {
        Node* copy = cur->next;
        cur->next = copy->next;        // 恢复原链表
        if (copy->next != nullptr)
            copy->next = copy->next->next;  // 接好新链表
        cur = cur->next;
    }
    
    return newHead;
}
```

时间 O(n)，空间 O(1)（不算输出）。
### ⑤ 面试常见追问

**Q：两种方法怎么选？**

哈希表法思路直观、好写、不易错，面试首选讲这个。交织法是 O(1) 空间的进阶优化，面试官追问空间优化时再讲。

**Q：交织法第二步为什么 `cur->next->random = cur->random->next`？**

`cur->next` 是 cur 的复制节点。`cur->random` 是原节点的 random 目标，而那个目标节点的下一个（`cur->random->next`）正好是它的复制节点。所以复制节点的 random 应该指向"原 random 目标的复制节点"，即 `cur->random->next`。

**Q：为什么交织法第三步要恢复原链表？**

因为我们破坏了原链表的 next（插入了复制节点）。如果不恢复，调用方拿到的原链表就乱了。这和回文链表恢复结构是同样的工程考量。
