## 朴素思路 vs 最优思路
**朴素思路：** 用哈希集合存链表A的所有节点，再遍历B，看哪个节点已经在集合里。能做，但要 O(m+n) 空间。
**最优思路：双指针，O(1) 空间**
关键洞察：**如果把两条链表首尾相连拼起来，长度就一样了。**

```
A的长度 = a（独有部分）+ c（公共部分）
B的长度 = b（独有部分）+ c（公共部分）

拼接：A走完接着走B = a + c + b
拼接：B走完接着走A = b + c + a

两者总长度相等！如果有交点，会在走了 a+b 步之后同时到达交点。
```
## 解题思路
1.最重要的解题思路就是双指针，分别走向的是headA与headB，如果二者分别从各自的初始位置开始遍历链表，当它们都遍历到各自的最后一个node后，它们再次遍历肯定就会到两个链表的交点处，若没有交点node那么就是都遍历到空指针了。
2.所以第一句就是自己判定二者指针是否同时指向空节点，若不是则就没有交点直接返回空指针。
3.两个指针的遍历的while循环，循环条件就是pA不能等于pB，如果说二者相等了要不就是找到了交点要不就是都指向了空指针，其中pA的遍历方法是正常遍历下去，但是如果pA遍历完了A链表或者pB遍历完了B链表，它们就得跳转到对方链表的第一个节点，所以遍历的方法如下
```
pA = (pA == nullptr) ? headB : pA->next; 

pB = (pB == nullptr) ? headA :pB->next;
```
4.最后返回结果随便是pA还是pB因为如果有交点，那它们指向的都是相交点


### ④ 最关键的两个细节

**为什么循环条件是 `pA != pB`，而不是判断值相等？**

链表节点比较应该比较**指针/地址**，不是值。题目特意提到："值为1的节点是不同节点"，这就是在提醒你不能用值判断，必须用指针判断是否是同一个节点。

**没有交点的情况会怎样？**

如果两条链表不相交，`pA` 和 `pB` 都会各走 `m+n` 步，最终同时变成 `nullptr`。这时候 `pA == pB == nullptr`，循环退出，返回 `nullptr`，正好符合题目要求。

---

### ⑤ 这道题为什么巧妙：把"长度不同"问题转化为"长度相同"

普通做法要先求出两条链表的长度差，让长的链表先走几步对齐，再一起走。**双指针的做法本质上做了同样的事，但通过"走完自己接着走对方"，巧妙地自动完成了长度对齐**，不需要额外计算长度。

### ⑥ 面试常见追问 + 答案

**Q：为什么两个指针一定会相遇？**

设A独有长度a，B独有长度b，公共部分长度c。pA总路程是 `a+c+b`，pB总路程是 `b+c+a`，两者相等。走到这个总路程时，两者必然在同一位置——如果有公共部分，相遇点正是交点；如果没有公共部分（c=0），两者同时到达 `nullptr`。

**Q：如果两条链表长度差很大，这个方法会不会变慢？**

不会，时间复杂度始终是 O(m+n)，和长度差无关，这正是这个技巧的优雅之处。

**Q：能不能用哈希集合做？复杂度怎样？**

可以，遍历A把所有节点地址存入哈希集合，再遍历B，第一个在集合中的节点就是交点。时间 O(m+n)，空间 O(m)，不如双指针法节省空间。
