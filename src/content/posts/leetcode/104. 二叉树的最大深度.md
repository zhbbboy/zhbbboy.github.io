### ① 核心痛点与破局点

- 🛑 **痛点**：一棵树有很多分支，可能有左子树很深，右子树很浅，也可能相反。如果从上往下数，很难一眼知道哪条路径是最长的。
    
- 🔄 **破局点**：**向下要答案，向上做决策。** * 不要站在根节点硬往下看，而是假设你的左右帮手已经帮你算好了左子树的最大深度和右子树的最大深度。
    
    - 作为根节点的你，只需要取它们之中的**较大值**，然后**加上自己这一层（+1）**，就是整棵树的最大深度。
        

### ② 整体思路：分治与递归

我们可以把算整棵树深度的大问题，拆解为算左右子树深度的小问题。

结合递归的两要素：

1. 🛑 **基线条件（Base Case）**：当遇到空节点 `nullptr` 时，深度是多少？
    
2. 🔄 **递归步骤（Recursive Step）**：
    
    - 递归计算左子树的最大深度 `leftDepth`。
        
    - 递归计算右子树的最大深度 `rightDepth`。
        
    - 返回两者之中的最大值加 1。
## 解题思路
1. 使用递归法
2. 终止条件：当访问到空节点的时候，说明是深度为0；返回0.
3. 递归步骤：将一颗树分为左子树和右子树分别去递归然后获取它们的最大深度
4. 最终决策：返回左子树和右子树二者中最大的深度然后再加上root这一层深度

### 完整代码

```
#include <algorithm>

class Solution {
public:
    int maxDepth(TreeNode* root) {
        // 要素 1：基线条件（出口）
        // 如果走到空节点，说明深度为 0
        if (root == nullptr) {
            return 0; 
        }

        // 要素 2：递归步骤（推进）
        // 分别计算左子树和右子树的最大深度
        int leftDepth = maxDepth(root->left);
        int rightDepth = maxDepth(root->right);

        // 最终决策：取左右深度的较大值，加上当前根节点这一层（+1）
        return std::max(leftDepth, rightDepth) + 1;
    }
};
```
