## 解题思路

## 一.递归法
### ① 核心痛点与破局点

- **痛点**：二叉树不是线性结构（不像数组或链表可以一路 `next` 到底），它有分叉（左子树和右子树）。面对这种多分支、层层嵌套的结构，如果硬用循环（迭代）去写，需要自己维护一个栈，代码极其容易写飞，难以一步看清核心逻辑。
    
- **破局点（回归递归的本质）**：**利用递归天然的“系统调用栈”**。二叉树本身就是一种递归定义的数据结构（一个节点，它的左右子树依然是二叉树）。
    
    - 无论是前序、中序还是后序遍历，它们的**本质区别仅仅是“访问根节点的时机”不同**，而整体的遍历骨架是完全一模一样的！
        

### ② 整体思路：中序遍历的“口诀”

中序遍历的访问顺序是：**左子树 $\rightarrow$ 根节点 $\rightarrow$ 右子树**。

我们直接套用你在开头学到的**递归两要素**：

1. **基线条件（Base Case/出口）**：当遍历到的节点是 `nullptr`（空节点）时，说明这条路走到头了，直接返回。
    
2. **递归步骤（Recursive Step/推进）**：
    
    - **先去左边**：递归调用左子树 `inorder(root->left)`。
        
    - **处理中间**：把当前节点的值 `root->val` 记录下来。
        
    - **再去右边**：递归调用右子树 `inorder(root->right)`。
        

### ③ 完整 C++ 代码（经典递归版）

面试时，为了避免全局变量带来的污染，通常会声明一个 `res` 数组，并通过一个私有辅助函数 `dfs` 来进行递归。

```C++
class Solution {
private:
    // 递归辅助函数
    void dfs(TreeNode* root, vector<int>& res) {
        // 要素 1：基线条件（出口）
        if (root == nullptr) {
            return;
        }

        // 要素 2：递归步骤（推进）
        dfs(root->left, res);      // 1. 先遍历左子树
        res.push_back(root->val);  // 2. 访问根节点（中序遍历的核心：在中间记录值）
        dfs(root->right, res);     // 3. 再遍历右子树
    }

public:
    vector<int> inorderTraversal(TreeNode* root) {
        vector<int> res;
        dfs(root, res);
        return res;
    }
};
```

### ④ 关键细节与面试追问

- **Q：如果面试官让我写“前序遍历”和“后序遍历”，代码怎么改？**
    
    - **A**：极其简单，只需要调整 `res.push_back(root->val)` 这行代码的**相对位置**：
        
        - **前序（根 $\rightarrow$ 左 $\rightarrow$ 右）**：先把值丢进 `res`，再递归左，最后递归右。
            
        - **后序（左 $\rightarrow$ 右 $\rightarrow$ 根）**：先递归左，再递归右，最后把值丢进 `res`。
            
- **Q：进阶要求（如题目左下角所示）：你能用迭代（非递归）的方法完成吗？**
    
    - **A**：面试时如果被追加这个问题，千万别慌。递归的本质是操作系统帮我们用了一个“隐式的栈”。迭代法就是**我们自己显式地用一个 `std::stack`**，先把所有左孩子一直压入栈，弹出一个节点并记录，再转向它的右孩子。
        

### ⑤ 避坑指南（面试挂点）

1. **空指针解引用**：在递归内部，进行 `root->left` 或 `root->right` 递归之前，**千万不要**提前去访问 `root->val`（除非已经过了 `root == nullptr` 的安全检查），否则面对空树 `[]` 时直接报段错误。
    
2. **辅助函数忘记传引用 `&`**：在 `dfs(TreeNode* root, vector<int>& res)` 中，`res` 必须传引用。如果写成传值（没加 `&`），每一层递归都会深度拷贝一个数组，不仅时间会退化，最后返回的 `res` 也是空的。

## 二.迭代法
### ① 核心痛点与破局点

- 🛑 **痛点**：中序遍历的顺序是“左 $\rightarrow$ 中 $\rightarrow$ 右”。当我们一路向左访问到最底层的左孩子时，我们需要“回头”去访问它的父亲节点。但二叉树的指针只有 `left` 和 `right`，没有指向父亲的 `parent` 指针。一旦走过头，就回不去了。
    
- 🔄 **破局点**：**用栈来当“后退记录仪”**。我们在往左走的过程中，把路过的所有节点都压入栈中。当左边走到头（遇到 `nullptr`）时，从栈顶弹出的节点，恰好就是当前最应该被访问的那个“父亲”或“左孩子”。
    

### ② 整体思路：指针一路向左，栈来负责回头

我们使用一个指针 `curr` 指向当前节点，和一个辅助栈 `st`：

1. **一路向左**：只要 `curr` 不为空，就把 `curr` 压入栈中，然后 `curr = curr->left`。直到把最左侧的一条边全部压入栈。
    
2. **弹出访问**：当 `curr` 为空时，说明左边到头了。此时从栈中弹出一个节点，记录它的值（这就是“中”）。
    
3. **转向右边**：因为中序是“左 $\rightarrow$ 中 $\rightarrow$ 右”，访问完中间后，立刻让 `curr = popped_node->right`，去处理右子树。
    
4. **循环条件**：只要 `curr` 不为空，或者栈内还有元素，就重复上述步骤。
    

### ③ 完整 C++ 代码（迭代版）
```C++
class Solution {
public:
    vector<int> inorderTraversal(TreeNode* root) {
        vector<int> res;
        stack<TreeNode*> st; // 手动维护的栈
        TreeNode* curr = root; // 当前正在遍历的指针

        // 只要当前节点不为空，或者栈里还有老路可以回，就继续循环
        while (curr != nullptr || !st.empty()) {
            // 1. 一路向左压栈
            while (curr != nullptr) {
                st.push(curr);
                curr = curr->left;
            }

            // 2. 左边到头了，弹出栈顶元素并访问
            curr = st.top();
            st.pop();
            res.push_back(curr->val); // 记录中间节点的值

            // 3. 转向右子树
            curr = curr->right;
        }

        return res;
    }
};
```
