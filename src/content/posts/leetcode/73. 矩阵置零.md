空间优化的三层思维过程：

- **第一层：O(mn) 额外空间（暴力）** —— 复制整个矩阵，扫描原矩阵，往副本里写 0。空间太大，不优雅。
- **第二层：O(m+n) 额外空间** —— 用两个数组 `rows[m]` 和 `cols[n]` 记录哪些行/列需要置零，再遍历矩阵统一处理。
- **第三层：O(1) 空间** —— 关键洞察：我们需要 `rows[m]` 和 `cols[n]` 来记录信息，但矩阵本身的第一行和第一列就有 m+n 个格子——直接借用它们来存！

## 解题思路（方法三：O(1) 空间）

1. 首先要明白矩阵的第一行与第一列是用来存储信息的：如果第 0 行第 1 列值为 0，应导致第 1 列全为 0；如果第 1 行第 0 列值为 0，那么第一行的所有值都得为 0。但有个特例是 `(0,0)`——它为 0 时分不清是第 0 行全为 0 还是第 0 列全为 0，所以索性约定 `(0,0)` 为 0 就代表第 0 行全为 0，另外定义 `col0` 来表示第一列是否全为 0。
2. 第一步先把所有需要置零的行/列找出来：遍历找出值为 0 的元素，例如下标 `(0,1)` 值为 0，就把 `(0,0)` 和 `(0,1)` 赋值为 0；若 `(1,0)` 值为 0，就令 `col0 = true` 并把 `(1,0)` 赋值为 0。由于 `(1,0)`、`(0,1)` 本身就是 0，不用多此一举。
3. 再根据第一行与第一列存储的标记来判断是否需要给该行/列的数赋值为 0。重要的点：因为第一行和第一列被用来存储标记，所以只能从右下角的矩阵元素开始（逆序）判断赋值。
4. 最后根据 `(0,0)` 和 `col0` 来判断是否要给第一行或第一列全赋值为 0。之所以最后处理，是因为前面几步都依赖第一行/第一列的标记，提前改会破坏标记。




### ④ 最容易出错的两个点

**为什么第二遍要从右下角往左上角（逆序）？**

因为第一行和第一列现在存的是"标记"，如果正序处理，可能先把第一行的标记格子置零，后续读到这个标记时发现是 0，但搞不清楚这个 0 是"原本有的0"还是"被我们刚才置的0"——顺序会污染标记。**逆序可以保证处理数据区时标记还没被动过。**

**为什么 j 从 1 开始而不是 0？**

第 0 列的信息已经存在 `col0` 变量里了，扫描时如果 `j=0` 且有 0，我们只更新 `col0`，不能更新 `matrix[i][0]`（否则会错误覆盖行标记）。

---

### ⑤ 面试常见追问 + 答案

**Q：为什么要单独用 `col0` 变量，不直接用 `matrix[0][0]`？**  
答：`matrix[0][0]` 被我们借来标记"第 0 行是否需要置零"。如果同时让它也标记"第 0 列是否需要置零"，两个信息混在一格里，无法区分。所以第 0 列的信息必须放到外面的变量里。

**Q：三种解法面试时应该先说哪个？**  
答：先说 O(m+n) 的方案展示思路清晰，主动提"还可以进一步优化到 O(1)"，然后讲 O(1) 方案。这样显示出你能从简单到复杂推进，思路有层次。


### 优化后的完整解题思路

> **第一步：提前记录第0列**  
> 遍历第0列，只要有任何元素为0，就令 `col0 = true`。因为 `[0][0]` 会被后续步骤征用来标记第0行，第0列的信息必须提前用独立变量保存。
> 
> **第二步：扫描矩阵，写入标记**  
> 遍历除第0列以外的所有元素（j 从1开始）。若 `matrix[i][j] == 0`，则令 `matrix[i][0] = 0`（标记第 i 行需要置零），令 `matrix[0][j] = 0`（标记第 j 列需要置零）。约定 `[0][0]` 专门用于标记第0行。
> 
> **第三步：从右下角逆序处理数据区**  
> 从 `i = m-1` 到 `i = 1`，`j = n-1` 到 `j = 1` 遍历。若 `matrix[i][0] == 0` 或 `matrix[0][j] == 0`，则将 `matrix[i][j]` 置零。必须逆序是因为第一行和第一列此时还存着标记，正序处理会把标记提前覆盖掉，导致后续读取错误。
> 
> **第四步：最后单独处理第0行和第0列**  
> 若 `matrix[0][0] == 0`，将第0行全部置零。若 `col0 == true`，将第0列全部置零。必须放在最后，因为前面几步都依赖第一行和第一列的标记值，提前修改会破坏标记。
