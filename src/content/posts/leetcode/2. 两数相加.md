### ① 破题：为什么逆序存储反而方便？

链表是逆序存的：`[2,4,3]` 表示数字 `342`。这个设计很巧妙——**逆序意味着链表头就是个位**，而我们做加法本来就是从个位开始算的，所以可以**从两个链表头开始，一边走一边加，自然产生进位往后传**，不需要任何反转。

```
342 + 465 = 807

   2 → 4 → 3   (个位 十位 百位)
 + 5 → 6 → 4
 ─────────────
   7 → 0 → 8
```

逐位相加：`2+5=7`，`4+6=10`（写0进1），`3+4+1=8`。完美对应竖式。

---

### ② 算法核心：carry 进位变量

设一个 `carry` 变量记录进位。每一位的计算公式：

```
当前位的和 = l1当前值 + l2当前值 + carry
新节点的值 = 和 % 10        （取个位）
新的carry  = 和 / 10        （取进位）
```

## 解题思路
1. 首先也创一个结果链表，然后建立一个指针来控制结果链表，然后创建一个进位符carry，若和大于10了，就carry为1。
2. 进入循环，循环的条件是若l1，l2，carry都为空才退出循环，否则则继续循环，定义x和y来存储l1和l2的val值，然后定义sum来存储和的结果。然后再加入新节点在结果链表末尾，其中的val为sum%10，然后tail前进，如果l1或l2都不为空则都前进
3. 返回dummy.next.


### ③ 完整 C++ 代码

cpp

```cpp
ListNode* addTwoNumbers(ListNode* l1, ListNode* l2) {
    ListNode dummy(0);          // 虚拟头节点
    ListNode* tail = &dummy;    // 结果链表的尾指针
    int carry = 0;              // 进位
    
    // 只要还有节点没处理完，或者还有进位，就继续
    while (l1 != nullptr || l2 != nullptr || carry != 0) {
        // 取出两个链表当前位的值，链表走完了就当作 0
        int x = (l1 != nullptr) ? l1->val : 0;
        int y = (l2 != nullptr) ? l2->val : 0;
        
        int sum = x + y + carry;
        carry = sum / 10;       // 计算新进位
        
        // 创建新节点存当前位（sum % 10）
        tail->next = new ListNode(sum % 10);
        tail = tail->next;
        
        // 两个指针各自前进（如果还没到末尾）
        if (l1 != nullptr) l1 = l1->next;
        if (l2 != nullptr) l2 = l2->next;
    }
    
    return dummy.next;
}
```

### ④ 三个关键设计

**循环条件为什么是 `l1 || l2 || carry`？**

三个条件缺一不可：

- `l1 || l2`：两条链表不等长，短的走完了长的还要继续
- `carry != 0`：处理示例3的情况，`999+1=1000`，最后还有一个进位要变成新的最高位节点

这三个条件用 `||` 连接，意思是"只要还有任何东西没处理完就继续"。

**为什么链表走完了当作 0？**

```cpp
int x = (l1 != nullptr) ? l1->val : 0;
```

两条链表不等长时，短链表后面相当于补 0。比如 `99 + 1`，个位 `9+1`，十位 `9+0`，把缺失的位当 0 处理，逻辑统一，不用单独写"处理剩余链表"的代码。

**dummy 虚拟头的作用**

和合并链表一样，省去对结果链表头节点的特殊处理，所有新节点都接在 `tail` 后面，最后返回 `dummy.next`。

### ⑤ 面试常见追问

**Q：为什么 carry 也要放进循环条件？不能循环外单独处理吗？**

可以单独处理，但放进循环条件更优雅。比如 `5+5=10`，两条链表都只有一位，循环跑一次后 l1、l2 都为 null，但 carry=1。如果循环条件没有 carry，就会漏掉最高位的 `1`。把 carry 放进条件，循环会再跑一次，`x=0, y=0, sum=0+0+1=1`，创建节点 `1`，正确。

**Q：时空复杂度？**

时间 O(max(m,n))，遍历较长的链表。空间 O(max(m,n))，结果链表的长度（不算输出的话是 O(1) 额外空间）。

**Q：如果链表是正序存储（链表头是最高位）怎么办（445题）？**

正序就不能直接从头加了，因为加法要从个位（最低位）开始。两种解法：先把两个链表反转，加完再反转回来；或者用栈，把两个链表的值分别压栈，再从栈顶（个位）开始弹出相加。
