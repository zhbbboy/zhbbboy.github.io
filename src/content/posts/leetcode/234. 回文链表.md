### ① 破题：链表的本质难点

数组判断回文很简单：左右双指针往中间走就行。但链表**只能从头往尾走，没法往回走**，这是核心矛盾。

解决思路：**把后半段链表反转，然后和前半段逐一比较。**

整体分四步：

```
① 快慢指针找中点
② 反转后半段链表  （直接复用 206 的代码）
③ 双指针从两端往中间比较
④ 恢复链表（面试加分，题目要求原始结构）
```

---

### ② 快慢指针找中点

快指针每次走 2 步，慢指针每次走 1 步，快指针到终点时慢指针正好在中间。

**奇偶链表的处理：**

```
奇数 [1,2,3,2,1]：slow 停在 3，后半段从 slow->next 开始
偶数 [1,2,2,1]：  slow 停在第一个 2，后半段从 slow->next 开始
```
## 解题思路
1. 首先还是得重新写一下翻转链表的方法，因为这题需要使用。
2. 先判断初始链表是否空链表或者是只有单个节点的链表，如果是的话则直接就是回文链表了
3. 先找链表中点，用快慢指针去寻找，也就是slow指针移动一步，fast指针移动两步，如果fast指针已经找到了最后个指针那么slow指针就指向了中点了（如果是奇数链表那就是中间节点，如果是偶数链表那就指向了前半部分链表的最后个节点）
4. 找到了中间节点，那就直接翻转后半部分的链表，并且将slow的next指针指向nullptr，这样就将一个链表分为两个链表了。
5. 用双指针p1，p2对两个链表进行比较，首先定义res为true，然后循环同步遍历两个链表若两个链表中有不一样val，那就将res改为false，并且break退出循环(其中循环的条件是p2的指针不为空，这是因为不管初始链表是奇数个节点还是偶数个节点，后半部分的节点数量都会小于等于前半部分的节点数量，如果初始链表是奇数个节点的话，那么slow指针指向的节点是不用比较的，但是它又在前半部分链表中，所以会比后半部分链表多一个节点，所以就按照后半部的链表作为循环条件)，恢复链表（也就是将slow->next指针重新指向翻转后的后半部分链表的head），再返回res



### ④ 两个容易出错的细节

**快慢指针的循环条件为什么是 `fast->next && fast->next->next`？**

需要保证 `fast->next->next` 访问安全，所以要先确认 `fast->next` 不为空，再确认 `fast->next->next` 不为空。如果写成 `fast && fast->next` 则 slow 会多走一步，中点位置偏右。
环检测：为了安全  
找中点：为了控制停留位置

**为什么以 `p2`（后半段）为循环终止条件，不用 `p1`？**

奇数长度链表（比如 `[1,2,3,2,1]`）中，前半段是 `[1,2,3]`，后半段是 `[2,1]`，前半段比后半段多一个中心节点 `3`。以 `p2` 为准，`p2` 走完时前半段的中间节点还没被访问，刚好跳过，逻辑正确。

### ⑤ 面试常见追问

**Q：为什么要恢复链表？**

题目说"函数结束后链表必须保持原始结构"。面试里考官会专门检查这一点，恢复体现了对使用方的尊重——调用者传进来一条链表，不应该被你静默修改掉结构。

**Q：时空复杂度？**

时间 O(n)，三次遍历（找中点 + 反转 + 比较），每次都是 O(n)。空间 O(1)，只用了几个指针变量，没有额外数组。

**Q：有没有更简单的写法？**

可以把链表值全部放入数组，再用双指针判断数组是否回文，代码更简单但空间 O(n)。面试时先说这个方案，再说 O(1) 空间的优化版本，展示思路层次。
