## 解题思路
1. 首先第一步先转置，其要点是只遍历上三角，不然容易重复swap了。
2. 其次就是左右翻转了。用reverse函数
3. 经过两次的变换就等于图形二元数组的旋转90°
4. 为什么转置+左右翻转 = 顺时针旋转90°？顺时针旋转：(i,j) → (j, n-1-i)，先转置： (i,j) → (j, i) 行列互换，再左右翻： (j,i) → (j, n-1-i) 列下标取镜像，两步合并： (i,j) → (j, n-1-i) 与旋转公式完全一致 ✓，逆时针旋转则改为：先左右翻，再转置（顺序反过来）



### ④ 转置时为什么 j 从 i+1 开始？

```
如果 j 从 0 开始，swap(matrix[i][j], matrix[j][i])
会先交换 (0,1) 和 (1,0)，
然后又交换 (1,0) 和 (0,1)，
等于换了两次，回到原样——什么都没做。
```

只遍历主对角线上方（`j > i`）就够了，每对元素只交换一次。

### ⑤ 面试追问 + 答案

**Q：如果是逆时针旋转 90° 怎么改？**

顺序反过来：先左右翻转，再转置。或者也可以先转置，再上下翻转。

cpp

```cpp
// 逆时针
// 方法：先左右翻，再转置
for (int i = 0; i < n; i++)
    reverse(matrix[i].begin(), matrix[i].end());
for (int i = 0; i < n; i++)
    for (int j = i + 1; j < n; j++)
        swap(matrix[i][j], matrix[j][i]);
```

**Q：旋转 180° 怎么做？**

做两次 90° 旋转，或者直接上下翻转再左右翻转：

cpp

```cpp
// 上下翻转
reverse(matrix.begin(), matrix.end());
// 再左右翻转每行
for (int i = 0; i < n; i++)
    reverse(matrix[i].begin(), matrix[i].end());
```

**Q：为什么这道题只适用于 n×n 方阵，如果是 m×n 矩形矩阵呢？**

转置后矩阵尺寸会从 m×n 变成 n×m，无法原地完成，需要额外空间。这道题限定了方阵正是为了让原地操作可行。
