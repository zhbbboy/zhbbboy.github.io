## ① 破题：题目要你干什么？

题目给你一个**升序数组**：

```
nums = [-10, -3, 0, 5, 9]
```

要你把它转换成一棵**高度平衡的二叉搜索树**。

---

## 什么是二叉搜索树？

二叉搜索树，也叫 BST。

它满足：

```
左子树所有节点 < 根节点 < 右子树所有节点
```

例如：

```
        0
       / \
     -3   9
     /   /
   -10  5
```

这就是一棵二叉搜索树。

因为：

```
-10 < -3 < 0 < 5 < 9
```

---

## 什么是高度平衡？

高度平衡的意思是：

```
左右子树高度差不能太大
```

这题不要求唯一答案，只要你构造出来的树是：

```
二叉搜索树 + 尽量平衡
```

就可以。
## ② 核心技巧：每次选中间元素作为根节点

因为数组已经升序排列。

如果你直接拿第一个元素当根：

```
-10 -> -3 -> 0 -> 5 -> 9
```

树会变成一条链，不平衡。

所以最好的办法是：

```
每次选择数组中间位置的元素作为根节点
```

例如：

```
nums = [-10, -3, 0, 5, 9]
```

中间元素是：

```
0
```

所以根节点是：

```
        0
```

然后：

```
左半部分 [-10, -3] 构造左子树
右半部分 [5, 9] 构造右子树
```

这就是递归。
## 解题思路
1. 首先是用的递归法，所以直接重新写了一个新的构建平衡二叉搜索树的函数，主要是方便于递归。
2. 函数含义：就是给三个形参，nums数组，要构建的数组左边界，要构建的数组右边界。直接就是根据数组构建二叉搜索树
3. 终止条件：如果left大于right，直接返回nullptr，然后找出mid，因为这样可以方便构建一个平衡的二叉树。然后要先new一个root节点，因为要构建树。
4. 信任递归，拿到子问题的结果：数组的左半部分用来构建左子树，右半部分的数组用来构建右子树
5. 最终返回root节点


## ④ 完整 C++ 代码

```cpp
class Solution {
public:
    TreeNode* sortedArrayToBST(vector<int>& nums) {
        return build(nums, 0, nums.size() - 1);
    }

    TreeNode* build(vector<int>& nums, int left, int right) {
        if (left > right) {
            return nullptr;
        }

        int mid = left + (right - left) / 2;

        TreeNode* root = new TreeNode(nums[mid]);

        root->left = build(nums, left, mid - 1);
        root->right = build(nums, mid + 1, right);

        return root;
    }
};
```

## ⑤ 关键设计解释

### 1. 为什么递归函数要有 left 和 right？

因为每次我们只处理数组中的一段。

```
build(nums, left, right)
```

表示：

```
用 nums[left...right] 这一段构造一棵平衡二叉搜索树
```

例如：

```
build(nums, 0, 4)
```

表示用整个数组建树。

```
build(nums, 0, 1)
```

表示用左半部分建左子树。

```
build(nums, 3, 4)
```

表示用右半部分建右子树。

---

### 2. 递归终止条件是什么？

```
if (left > right) {
    return nullptr;
}
```

意思是：

```
当前区间已经没有元素了，返回空节点。
```

比如：

```
build(nums, 0, -1)
```

说明这段区间不存在。

所以返回：

```
nullptr
```

---

### 3. 为什么选 mid 作为根？

因为数组是有序的。

如果选中间元素作为根：

```
左边元素都比根小
右边元素都比根大
```

所以天然满足二叉搜索树性质。

同时，左右两边元素数量接近，所以树会比较平衡。

---

### 4. 为什么左子树是 left 到 mid - 1？

因为：

```
nums[mid]
```

已经被拿来当根节点了。

它左边的元素都比它小：

```
nums[left ... mid - 1]
```

所以构造左子树。

---

### 5. 为什么右子树是 mid + 1 到 right？

同理，右边元素都比根节点大：

```
nums[mid + 1 ... right]
```

所以构造右子树。

# ⑥ 递归三要素套这题

你之前问过递归三要素，这题正好可以套进去。

## 1. 函数定义

```
build(nums, left, right)
```

表示：

```
把 nums[left...right] 这一段有序数组转换成平衡 BST，并返回根节点。
```

---

## 2. 终止条件

```
if (left > right) return nullptr;
```

区间为空，没有节点。

---

## 3. 当前层做什么？

当前层只做三件事：

```
1. 取中间元素作为根节点
2. 递归构造左子树
3. 递归构造右子树
```

代码就是：

```
int mid = left + (right - left) / 2;

TreeNode* root = new TreeNode(nums[mid]);

root->left = build(nums, left, mid - 1);
root->right = build(nums, mid + 1, right);

return root;
```

# ⑦ 面试常见追问

## Q1：为什么不能直接从左到右插入 BST？

如果你从左到右插入：

```
[-10, -3, 0, 5, 9]
```

树会变成：

```
-10
   \
   -3
      \
       0
        \
         5
          \
           9
```

这是一条链，不平衡。

所以必须每次选中间元素。

---

## Q2：mid 可以选左中点还是右中点吗？

可以。

比如长度为偶数：

```
[1, 3]
```

你可以选 `1` 当根：

```
1
 \
  3
```

也可以选 `3` 当根：

```
  3
 /
1
```

这两个都是高度平衡 BST。

所以代码里：

```
int mid = left + (right - left) / 2;
```

选的是偏左中点。

也可以写成：

```
int mid = left + (right - left + 1) / 2;
```

选偏右中点。

都能通过。

---

## Q3：为什么 `mid` 不直接写 `(left + right) / 2`？

可以写，但更推荐：

```
int mid = left + (right - left) / 2;
```

因为如果 `left + right` 很大，理论上可能整数溢出。

虽然这题数据规模不大，但面试里推荐写更安全的形式。

# ⑧ 复习版总结

```
108. 将有序数组转换为二叉搜索树

解题思路：递归 + 二分。

1. 有序数组转换为 BST，关键是利用有序性：
   左边元素小于根，右边元素大于根。

2. 为了让树高度平衡，每次选择当前区间的中间元素作为根节点。

3. 定义递归函数 build(nums, left, right)：
   表示用 nums[left...right] 构造一棵平衡 BST，并返回根节点。

4. 递归终止条件：
   如果 left > right，说明区间为空，返回 nullptr。

5. 当前层逻辑：
   - mid = left + (right - left) / 2
   - nums[mid] 创建根节点
   - 左半区间 [left, mid - 1] 构造左子树
   - 右半区间 [mid + 1, right] 构造右子树
   - 返回 root

核心记忆：
有序数组建平衡 BST = 每次取中点做根，左边建左子树，右边建右子树。
```

这题本质上是二叉树递归入门题。它和二分查找很像，只不过二分查找是“找元素”，这题是“用中点建树”。
