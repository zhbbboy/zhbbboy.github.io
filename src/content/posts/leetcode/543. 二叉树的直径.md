这道题是二叉树 DFS 的经典入门题，也是面试高频题，核心考察**后序遍历的妙用**和对 “树的深度 / 直径” 概念的理解。

## 一、题目彻底理解

### 1. 题目定义

二叉树的**直径**：树中**任意两个节点之间最长路径的长度**，路径长度用「边的数量」计算。

- 关键点：这条路径**可以不经过根节点**，这是最容易踩坑的地方。
	例如如下：
	我们构造一棵 “头重脚轻” 的树：根节点只有左子树很茂盛，右子树几乎没有。
	
	
	
	```plaintext
	        1  ← 整棵树的根节点
	       /
	      2
	     / \
	    3   6
	   /     \
	  4       7
	 /
	5
	```
	
	现在我们分别算两条路径的长度（边数）：
	
	1. **经过根节点 1 的最长路径**
	    
	    根节点 1 的右子树是空的，没有节点；左子树最深能走到节点 5。
	    
	    所以经过根的最长路径就是 `1→2→3→4→5`，一共 **4 条边**。
	    
	2. **不经过根节点 1 的路径**
	    
	    我们以节点 2 为最高点，拼接它的左右两条分支：`5→4→3→2→6→7`。
	    
	    这条路径一共 **5 条边**，比经过根的路径更长。
	    
	
	因此这棵树的直径是 5，而这条最长路径**全程都在根节点的左子树内部，根本没有经过根节点 1**。
	

路径长度 = 路径上的节点数 - 1。比如路径 `4→2→1→3` 有 4 个节点，长度就是 3 条边。
## 解题思路
### 一.递归法
1. 终止条件：如果节点为空则返回深度为0；
2. 后序遍历：先算左子树的速度再算右子树深度
3. 更新直径：拿到左右深度后，当前节点能 “拼出” 的最长路径就是左右深度之和，用它更新全局最大值。
4. 返回深度：父节点只能选当前节点的一条分支来拼接路径，所以返回「左右深度的较大值 + 1」。
#### 递归法完整实现如下：

```cpp
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode() : val(0), left(nullptr), right(nullptr) {}
 *     TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
 *     TreeNode(int x, TreeNode *left, TreeNode *right) : val(x), left(left), right(right) {}
 * };
 */
class Solution {
private:
    int maxDiameter = 0; // 全局变量，记录最大直径

    // 函数作用：返回以node为根的子树的最大深度（边数）
    int dfs(TreeNode* node) {
        // 终止条件：空节点深度为0
        if (node == nullptr) {
            return 0;
        }

        // 后序遍历：先递归左右子树，拿到左右子树的深度
        int leftDepth = dfs(node->left);
        int rightDepth = dfs(node->right);

        // 【关键】在当前节点计算直径：左深度 + 右深度
        // 因为深度是边数，直接相加就是经过当前节点的路径边数
        maxDiameter = max(maxDiameter, leftDepth + rightDepth);

        // 返回给父节点：当前子树的最大深度 = 左右更深的那个 + 1（当前节点到子节点的一条边）
        return max(leftDepth, rightDepth) + 1;
    }

public:
    int diameterOfBinaryTree(TreeNode* root) {
        dfs(root);
        return maxDiameter;
    }
};
```

### 复杂度分析

- **时间复杂度：O (n)**
    
    每个节点只会被访问一次，每个节点内的操作都是 O (1)，整体线性时间。
- **空间复杂度：O (h)**
    
    h 是树的高度，由递归调用栈占用。
    
    - 平衡二叉树：h = logn，空间 O (logn)
    - 最坏情况（链状树）：h = n，空间 O (n)
### 高频易错点

1. **混淆 “节点数” 和 “边数”**
    
    很多人会写成 `leftDepth + rightDepth + 1`，这是算成了节点数。题目要求边数，直接相加即可，千万不要多加 1。
2. **误以为直径一定经过根节点**
    
    这是最常见的错误。最长路径完全可能出现在某一棵子树内部，所以必须遍历所有节点更新最大值。
3. **递归函数职责混淆**
    
    记住：返回值是给父节点用的 “深度”，直径是在内部偷偷算的，不要把直径当返回值返回给上层。

### 拓展：不用全局变量的写法（分治思想）

如果面试中不允许用全局变量，可以让递归函数返回一个「深度 + 当前子树最大直径」的二元组，本质是纯函数式的分治思路：

```cpp
class Solution {
    // 返回值：first = 子树深度，second = 子树内的最大直径
    pair<int, int> dfs(TreeNode* node) {
        if (!node) return {0, 0};
        auto left = dfs(node->left);
        auto right = dfs(node->right);
        
        int depth = max(left.first, right.first) + 1;
        int curDiameter = left.first + right.first;
        int maxD = max({curDiameter, left.second, right.second});
        return {depth, maxD};
    }
public:
    int diameterOfBinaryTree(TreeNode* root) {
        return dfs(root).second;
    }
};
```

这种写法没有副作用，状态更清晰，也是算法设计里的经典思路。


### 二.迭代法
递归版的核心是**后序遍历**：先处理完左右子树，拿到它们的深度，再计算当前节点的直径并返回深度。

迭代版就是把「系统调用栈」换成「手动栈」，同时解决两个问题：

1. 怎么保证**左右子树都处理完后，再处理当前节点**（后序顺序）；
2. 怎么保存每个节点的深度，供父节点读取（替代递归的返回值）。

业界的写法是**栈 + 访问标记法**：

- 栈里存 `(节点, 是否已访问)`，`false` 表示第一次入栈，只做 “压入子节点” 的操作；
- `true` 表示第二次弹出，此时左右子树已经全部处理完毕，可以计算当前节点的深度和直径；
- 用一个哈希表 `depthMap` 缓存每个节点的深度，父节点可以直接查表获取子节点深度。

### 完整 C++ 迭代实现

```cpp
class Solution {
public:
    int diameterOfBinaryTree(TreeNode* root) {
        if (root == nullptr) return 0;

        int maxDiameter = 0;
        // 栈元素：(节点指针, 是否已访问/处理)
        stack<pair<TreeNode*, bool>> stk;
        // 缓存每个节点对应的子树最大深度
        unordered_map<TreeNode*, int> depthMap;

        stk.push({root, false});

        while (!stk.empty()) {
            auto [node, visited] = stk.top();
            stk.pop();

            if (!visited) {
                // 第一次访问：先把自己标记为已访问，重新压回栈
                stk.push({node, true});
                // 注意：栈是后进先出，所以先压右孩子，再压左孩子
                // 保证弹出顺序是 左 -> 右 -> 根，符合后序遍历
                if (node->right != nullptr) {
                    stk.push({node->right, false});
                }
                if (node->left != nullptr) {
                    stk.push({node->left, false});
                }
            } else {
                // 第二次访问：左右子树都处理完了，计算当前节点的深度和直径
                int leftDepth = 0;
                int rightDepth = 0;
                // 从哈希表读取左右子树的深度，空节点默认深度为0
                if (node->left != nullptr) {
                    leftDepth = depthMap[node->left];
                }
                if (node->right != nullptr) {
                    rightDepth = depthMap[node->right];
                }

                // 更新全局最大直径
                maxDiameter = max(maxDiameter, leftDepth + rightDepth);
                // 计算当前节点的深度，存入哈希表，供父节点使用
                depthMap[node] = max(leftDepth, rightDepth) + 1;
            }
        }

        return maxDiameter;
    }
};
```

#### 1. 初始化部分

- 边界判断：根节点为空直接返回 0；
- `maxDiameter`：和递归版一样，记录全局最大直径；
- `stk`：手动模拟递归调用栈，`bool` 标记用来区分 “第一次压栈” 和 “第二次处理”；
- `depthMap`：替代递归函数的返回值，把每个节点的深度存起来，父节点处理时直接查表。

#### 2. 第一次弹出（visited = false）

对应递归里「进入函数，先递归调用左右子树」的阶段：

1. 把当前节点重新标记为 `true` 压回栈 —— 相当于 “等左右子树都跑完了，再回来处理我”；
2. **先压右孩子，再压左孩子**：因为栈是后进先出，这样下一轮弹出的会是左孩子，保证左子树先处理，和递归顺序完全一致。

#### 3. 第二次弹出（visited = true）

对应递归里「左右递归都返回了，处理当前节点」的阶段：

1. 从哈希表取出左右子树的深度（空节点深度为 0）；
2. 用 `左深度 + 右深度` 更新最大直径；
3. 计算当前节点深度 = `max(左深, 右深) + 1`，存入哈希表，留给父节点查表使用。

这一步做的事情，和递归版 `dfs` 函数里递归调用之后的代码完全等价。

### 用示例 1 走一遍执行流程

以题目示例 1 为例：
```
    1
   / \
  2   3
 / \
4   5
```

1. 压入 `(1, false)`；
2. 弹出 `(1, false)` → 压入 `(1, true)`，再压 `(3, false)`、`(2, false)`；
3. 弹出 `(2, false)` → 压入 `(2, true)`，再压 `(5, false)`、`(4, false)`；
4. 弹出 `(4, false)` → 压入 `(4, true)`，无左右孩子；
5. 弹出 `(4, true)` → 左右深度都是 0，直径 0，`depthMap[4] = 1`；
6. 弹出 `(5, false)` → 压入 `(5, true)`，无左右孩子；
7. 弹出 `(5, true)` → 左右深度都是 0，直径 0，`depthMap[5] = 1`；
8. 弹出 `(2, true)` → 左深 1，右深 1，直径更新为 2，`depthMap[2] = 2`；
9. 弹出 `(3, false)` → 压入 `(3, true)`，无左右孩子；
10. 弹出 `(3, true)` → 左右深度都是 0，直径保持 2，`depthMap[3] = 1`；
11. 弹出 `(1, true)` → 左深 2，右深 1，直径更新为 3，`depthMap[1] = 3`；
12. 栈空，返回结果 3。

整个过程和递归的执行顺序、计算逻辑完全一致，只是把系统栈换成了手动栈，把返回值存在了哈希表里。

### 复杂度与注意事项

1. **时间复杂度：O (n)**
    
    每个节点入栈、出栈各 2 次，哈希表读写都是 O (1)，整体仍是线性时间。
2. **空间复杂度：O (n)**
    
    栈和哈希表最坏情况（链状树）都占用 O (n) 空间；平衡树时为 O (logn)。
3. **写法说明**
    
    这是对应递归逻辑的迭代后序写法，面试中如果被要求写非递归，这种写法思路清晰，不容易出错。
    
    实际工程和面试中，递归写法已经足够；迭代写法主要用于理解递归本质，或是应对节点极多、可能栈溢出的极端场景。
