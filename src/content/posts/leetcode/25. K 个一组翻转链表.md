### ① 破题：分组 + 反转 + 拼接

把链表每 K 个分成一组，对每组做反转。核心难点有三个：

第一，反转前要先**确认这一组够不够 K 个**（不够就保持原序，直接返回）。  
第二，反转一组后，要把它和**前一组、后一组正确拼接**起来。  
第三，反转操作本身复用 206 的技巧，但要限定在 K 个节点的范围内。

## 解题思路
### 一.递归法
用你刚学的递归五步来想：

**语义：** `reverseKGroup(head, k)` 返回"从 head 开始、每 K 个一组翻转后的新头"。

**终止条件：** 如果从 head 开始不足 K 个节点，直接返回 head（保持原序）。

**当前层：** 反转前 K 个节点，然后让反转后的尾节点（原来的 head）接到"剩余部分递归处理的结果"。

```cpp
ListNode* reverseKGroup(ListNode* head, int k) {
    // 第一步：检查是否够 k 个节点
    ListNode* node = head;
    for (int i = 0; i < k; i++) {
        if (node == nullptr) return head;  // 不足 k 个，保持原序返回
        node = node->next;
    }
    // 此时 node 指向第 k+1 个节点（下一组的头）
    
    // 第二步：反转前 k 个节点（206 的逻辑，限定范围）
    ListNode* prev = node;  // 关键：prev 初始化为下一组的头，反转后第一个节点直接接上下一组
    ListNode* curr = head;
    for (int i = 0; i < k; i++) {
        ListNode* next = curr->next;
        curr->next = prev;
        prev = curr;
        curr = next;
    }
    // 反转后，prev 是这一组的新头，head 是这一组的新尾
    
    // 第三步：head（新尾）接到"剩余部分递归的结果"
    head->next = reverseKGroup(node, k);
    
    return prev;  // 返回这一组反转后的新头
}
```

---

### ④ 递归法的精妙之处

**`prev` 初始化为 `node`（下一组的头），而不是 `nullptr`，为什么？**

普通反转里 `prev` 初始为 `nullptr`，因为反转后第一个节点是尾节点，指向 null。但这里反转的是一组，反转后的尾节点（原 head）应该接到下一组。

巧妙之处在于：把 `prev` 初始化为 `node`（下一组的头），反转过程中第一次执行 `curr->next = prev` 时，原 head 就直接指向了下一组的头。虽然第三步又用 `head->next = reverseKGroup(node, k)` 重新接了一次（因为下一组也要被反转），但这个初始化保证了即使下一组不足 k 不反转时，连接也是正确的。

**反转后 `head` 为什么变成了新尾？**

反转一组后，原来的第一个节点（head）跑到了最后，所以 head 现在是这组的尾节点，它负责连接下一组。

### ⑤ 面试常见追问

**Q：时空复杂度？**

时间 O(n)，每个节点被遍历常数次（检查一次 + 反转一次）。递归法空间 O(n/k)（递归深度是组数）；迭代法可以做到 O(1) 空间，是进阶要求的最优解。

**Q：迭代法怎么写？**

迭代法用 dummy 头，外层循环每次定位一组的边界（找到第 k 个节点），反转这组，再把前驱接到新头、新尾接到下一组。逻辑和递归一致但要手动维护组间连接，代码更长更容易错，建议先掌握递归。

**Q：和 24 题什么关系？**

24 题就是 K=2 的特例。掌握了这道题，把 k 设成 2 就是 24 题。

### 二.迭代法
### ① 迭代法的整体框架

用一个 `dummy` 虚拟头，外层循环逐组处理。每一组需要维护一个关键指针 `prevGroupTail`——**上一组的尾节点**（也就是当前组的前驱），它负责把上一组和当前组反转后的新头连起来。

每一轮做四件事：

```
① 从 prevGroupTail 往后数 k 个，确认这一组够不够 k 个节点
② 反转这 k 个节点
③ 把上一组的尾，接到这一组反转后的新头
④ 更新 prevGroupTail 到这一组反转后的尾，准备下一组
```

### ③ 迭代法完整代码

cpp

```cpp
ListNode* reverseKGroup(ListNode* head, int k) {
    ListNode dummy(0);
    dummy.next = head;
    ListNode* prevTail = &dummy;  // 上一组的尾（当前组的前驱）
    
    while (true) {
        // ① 从 prevTail 往后数 k 个，找这一组的最后一个节点 groupEnd
        ListNode* groupEnd = prevTail;
        for (int i = 0; i < k; i++) {
            groupEnd = groupEnd->next;
            if (groupEnd == nullptr) return dummy.next;  // 不足 k 个，结束
        }
        
        // 记录关键节点
        ListNode* groupStart = prevTail->next;   // 本组第一个（反转后变尾）
        ListNode* nextStart = groupEnd->next;    // 下一组第一个（先存好）
        
        // ② 反转本组 k 个节点：[groupStart, groupEnd]
        // 标准反转，prev 初始化为 nextStart，让 groupStart 直接接到下一组
        ListNode* prev = nextStart;
        ListNode* curr = groupStart;
        while (curr != nextStart) {
            ListNode* next = curr->next;
            curr->next = prev;
            prev = curr;
            curr = next;
        }
        // 反转后：groupEnd 是新头，groupStart 是新尾
        
        // ③ 上一组的尾接到本组新头（groupEnd）
        prevTail->next = groupEnd;
        
        // ④ 更新 prevTail 到本组新尾（groupStart），处理下一组
        prevTail = groupStart;
    }
}
```

---

### ④ 三个最容易踩坑的细节

**反转的循环条件为什么是 `curr != nextStart`，不是 `curr != nullptr`？**

普通反转走到链表末尾（nullptr）停。但这里只反转一组（k 个），不是整条链表。`nextStart` 是这一组之后的第一个节点，是这一组的"边界"。当 `curr` 走到 `nextStart` 就说明这一组的 k 个节点反转完了，必须停下，否则会把后面的组也反转了。

**`prev` 为什么初始化为 `nextStart` 而不是 `nullptr`？**

和递归法同理。反转后 groupStart 变成本组尾节点，它应该接到下一组的头（nextStart）。把 prev 初始化为 nextStart，反转时第一次执行 `curr->next = prev` 就让 groupStart 直接指向了下一组，省去了第③步之后再手动连接 groupStart 的麻烦。

**为什么 `nextStart` 要在反转前提前存好？**

反转过程会修改 `groupEnd->next`（它会指向组内前一个节点），一旦反转开始，就找不到下一组的头了。所以必须在反转前用 `groupEnd->next` 把 nextStart 抢救出来。这和反转链表里"先存 next"是同一个道理。

### ⑤ 迭代 vs 递归对比

||递归法|迭代法|
|---|---|---|
|思路|反转一组 + 信任剩余递归处理|用 prevTail 手动维护组间连接，逐组循环|
|空间|O(n/k)，递归栈深度 = 组数|O(1)，只用几个指针|
|代码|简洁，易理解|较长，指针多，易错|
|面试|先讲递归展示思路|进阶要求 O(1) 空间时给迭代|

**面试建议：** 先用递归讲清楚思路（反转一组 + 处理剩余），面试官如果追问 O(1) 空间，再写迭代法。两个都能写说明你掌握得透彻。
