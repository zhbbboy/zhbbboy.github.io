### ① 破题：交换的本质是改指针

要把 `1→2→3→4` 变成 `2→1→4→3`，对每一对相邻节点（1和2、3和4），都要把它们的指向反过来，同时保证前后两对能正确连接。

关键是要有一个**前驱节点（prev）**，因为交换一对节点后，需要让前驱指向新的第一个节点。这就是 dummy 的用武之地。
## 解题思路
### 一. 迭代法：三个指针协作

每次处理一对节点，需要操作三个连接点。设 `prev` 是这对节点的前驱，`first` 和 `second` 是要交换的一对：

```
交换前：prev → first → second → 后续
交换后：prev → second → first → 后续
```

需要改三根指针：

```
① prev->next = second      （前驱指向 second）
② first->next = second->next （first 接到后续）
③ second->next = first      （second 指回 first）
```
![](Pasted%20image%2020260625124323.png)

1. 先定义头节点dummy，其next指针指向head。定义prev指针指向头节点。
2. 进入交换循环，首先必须得有两个节点去交换，所以prev->next和prev->next->next不能为空，否则就退出循环了。定义一个first指针指向要交换的第一个数，定义一个second指针指向第二个要交换的数，然后再进行三步交换。交换完了后将prev指针指向first指针，prev 移动到这一对的末尾（现在是 first），准备处理下一对。
### ③ 迭代法完整代码

```cpp
ListNode* swapPairs(ListNode* head) {
    ListNode dummy(0);
    dummy.next = head;
    ListNode* prev = &dummy;
    
    // 只要后面还有两个节点可以交换
    while (prev->next != nullptr && prev->next->next != nullptr) {
        ListNode* first = prev->next;
        ListNode* second = prev->next->next;
        
        // 三步交换
        prev->next = second;          // ① 前驱指向 second
        first->next = second->next;   // ② first 接到后续
        second->next = first;         // ③ second 指回 first
        
        // prev 移动到这一对的末尾（现在是 first），准备处理下一对
        prev = first;
    }
    
    return dummy.next;
}
```

### ④ 几个细节

**循环条件为什么是 `prev->next && prev->next->next`？**

要交换一对，必须确保 prev 后面至少有两个节点。如果只剩一个节点（奇数长度的最后一个），就不交换，保持原样。两个条件保证了访问安全且有完整一对可换。

**三步顺序能不能乱？**

不能。第②步 `first->next = second->next` 必须在第③步之前，因为 `second->next` 原本指向后续节点（如3），如果先执行③把 `second->next` 改成 first，就丢失了后续节点的位置。这和反转链表里"先存住 next"是同样的道理。

**为什么 prev 最后移到 first 而不是 second？**

交换后顺序变成 `second → first`，first 现在是这一对的最后一个节点，它就是下一对的前驱。所以 prev 移到 first。

### 二.递归法
1. 终止条件：head为空或者head->next为空，然后返回头节点，当前要交换的一对：head 和 head->next
2. 信任递归：交换后面的部分，返回的是后面处理完的头，head->next 应该接到"剩余部分交换后的头"
3. second 指回 head，完成当前这一对的交换，交换后 second 是新的头，返回它

用递归三要素来想：

```cpp
ListNode* swapPairs(ListNode* head) {
    // 终止条件：空或只剩一个节点，无法交换，直接返回
    if (head == nullptr || head->next == nullptr) {
        return head;
    }
    
    // 当前要交换的一对：head 和 head->next
    ListNode* second = head->next;
    
    // 信任递归：交换后面的部分，返回的是后面处理完的头
    // head->next 应该接到"剩余部分交换后的头"
    head->next = swapPairs(second->next);
    
    // second 指回 head，完成当前这一对的交换
    second->next = head;
    
    // 交换后 second 是新的头，返回它
    return second;
}
```

**用递归语义读：** `swapPairs(head)` 返回"从 head 开始两两交换后的新头"。当前层只处理 head 和 second 这一对：让 head 接到"后面交换好的结果"，再让 second 指回 head，返回 second 作为新头。剩下的交给递归。

### ⑥ 面试常见追问

**Q：题目为什么强调"不能修改节点内部的值"？**

如果允许改值，直接交换相邻两个节点的 `val` 就行，太简单了。题目要求交换节点本身（改指针），是为了考察你对指针操作的掌握，这在实际工程中更有意义（节点可能携带大量数据，改指针比拷贝整个节点高效）。

**Q：迭代和递归哪个好？**

时间都是 O(n)。迭代空间 O(1)，递归空间 O(n)（栈深度）。面试两种都要会，迭代是更优解，但递归代码更简洁优雅。
