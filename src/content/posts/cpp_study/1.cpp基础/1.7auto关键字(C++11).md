## auto简介

在 C++98/03 中，`auto` 曾是一个几乎无人问津的关键字，用来声明变量具有“自动生存期”（由于局部变量默认就是自动生存期，所以它完全是多余的）。

从 C++11 开始，引入了现代 C++（Modern C++）的概念，`auto` 被赋予了全新的使命——**自动类型推导**。它让编译器在编译期通过变量的显式初始化表达式，自动帮我们推断出变量的实际类型。作为一个新的类型指示符来指示编译器，auto声明的变量必须由编译器在编译时期推导而得。
```cpp
#include <iostream>
using namespace std;
double Fun()
{
	return 3.14;
}
int main()
{
	int a = 10;
	auto b = a;
	auto c = 'A';
	auto d = Fun();
	//打印变量b,c,d的类型
	cout << typeid(b).name() << endl;//打印结果为int
	cout << typeid(c).name() << endl;//打印结果为char
	cout << typeid(d).name() << endl;//打印结果为double
	return 0;
}
```
**注意**：使用auto变量时必须对其进行初始化，在编译阶段编译器需要根据初始化表达式来推导auto的实际类型。因此，auto并非是一种“类型”的声明，而是一个类型声明的“占位符”，编译器在编译期会将auto替换为变量实际的类型。

## auto的使用细则
#### 一、auto与指针和引用结合起来使用
 用auto声明指针类型时，用auto和auto* 没有任何区别，但用auto声明引用类型时必须加&。
```cpp
#include <iostream>
using namespace std;
int main()
{
	int a = 10;
	auto b = &a;   //自动推导出b的类型为int*
	auto* c = &a;  //自动推导出c的类型为int*
	auto& d = a;   //自动推导出d的类型为int
	//打印变量b,c,d的类型
	cout << typeid(b).name() << endl;//打印结果为int*
	cout << typeid(c).name() << endl;//打印结果为int*
	cout << typeid(d).name() << endl;//打印结果为int
	return 0;
}
```
**注意：用auto声明引用时必须加&，否则创建的只是与实体类型相同的普通变量。**
#### 二、在同一行定义多个变量
当在同一行声明多个变量时，这些变量必须是相同的类型，否则编译器将会报错，因为编译器实际只对第一个类型进行推导，然后用推导出来的类型定义其他变量。
```cpp
int main()
{
	auto a = 1, b = 2; //正确
	auto c = 3, d = 4.0; //编译器报错：“auto”必须始终推导为同一类型
	return 0;
}

```
## auto不能推导的场景

#### 一、auto不能作为函数的参数
以下代码编译失败，auto不能作为形参类型，因为编译器无法对x的实际类型进行推导。

```cpp
void TestAuto(auto x)
{}
```
#### 二、auto不能直接用来声明数组

```cpp
int main()
{
	int a[] = { 1, 2, 3 };
	auto b[] = { 4, 5, 6 };//error
	return 0;
}
```




## 1. 为什么需要 auto？（解决核心痛点）

`auto` 的引入绝不仅仅是为了“偷懒少敲键盘”，它核心解决了两个问题：**可读性灾难** 和 **强耦合**。

### 场景一：简化冗长复杂的类型（提升可读性）

在引入 `auto` 之前，遍历一个标准库容器（如 `std::map`）的迭代器代码简直是噩梦：
```C++
#include <map>
#include <string>

std::map<std::string, std::map<int, double>> myComplexMap;

// C++98: 类型长到崩溃
std::map<std::string, std::map<int, double>>::const_iterator it = myComplexMap.cbegin();

// C++11 之后: 瞬间清爽
auto it = myComplexMap.cbegin(); 
```

### 场景二：应对无法提前知晓的临时类型（如 Lambda 表达式）

现代 C++ 中广泛使用的 Lambda 表达式（匿名函数），其底层类型是由编译器在编译期动态生成的独一无二的“闭包类型”。人类程序员是无法直接写出那个类型的，必须依赖 `auto` 来接收：

```C++
auto myLambda = [](int a, int b) { return a + b; };
```

## 2. auto 的类型推导规则（核心铁律）

`auto` 并不是万能的魔法，它在推导时有一套严格的、与函数模板类型推导完全一致的规则。

### 规则一：auto 默认会“剥离” const 属性和引用属性

当直接使用 `auto` 声明变量时，它会丢弃原对象的 `const` 限制和 `&`（引用）标记。

```C++
int x = 10;
const int cx = x;
const int& rx = x;

auto a = cx; // a 的类型是 int（const 被抛弃，变成一份独立拷贝）
auto b = rx; // b 的类型是 int（引用和 const 都被抛弃）
```

### 规则二：如果想保留 const 或引用，必须显式加上修饰符

如果你不希望发生对象拷贝，而是想定义一个引用，或者想保留常量只读属性，你需要给 `auto` 穿上“外衣”：

```C++
auto& c = cx;       // c 的类型是 const int&（引用保留了 cx 的 const 属性）
const auto d = x;   // d 的类型是 const int
```

### 规则三：万能的复合形态——“auto&&”

在现代 C++ 的高级代码（如泛型编程、完美转发）中，经常能看到 `auto&&`。

- 这里的 `&&` 并不单纯代表右值引用，它被称为**万能引用**。
    
- 它极其强大：如果你给它传左值，它就推导为左值引用；传右值，它就推导为右值引用。

## 3. auto 在现代 C++ 中的进阶演进

随着 C++ 标准的不断迭代，`auto` 的应用范围变得越来越广：

### 1. C++14：支持函数返回值推导

从 C++14 开始，函数的返回值也可以让编译器自己去猜了：

```C++
auto add(int a, double b) {
    return a + b; // 编译器自动推导返回值类型为 double
}
```

### 2. C++14：泛型 Lambda 表达式

允许在 Lambda 表达式的参数列表中直接使用 `auto`，这使得 Lambda 本质上变成了一个微型的函数模板：
```C++
auto printLog = [](const auto& data) {
    std::cout << data << std::endl;
};
printLog(123);     // 正确
printLog("hello"); // 正确
```
### 3. C++20：捷径函数模板（Abbreviated Function Templates）

从 C++20 开始，普通的普通函数参数也可以直接写 `auto` 了，它在底层会被自动展开为模板函数（Template）：
```C++
// C++20 允许：等价于 template<typename T> void process(T x);
void process(auto x) { 
    // ... 
}
```

## 4. auto 的限制（什么时候不能用）

虽然 `auto` 很聪明，但在以下几种情况下，编译器会罢工：

1. **不能用于非静态成员变量的声明**：

    ```C++
    class Person {
        auto age = 18; // ❌ 错误！类成员变量的内存布局必须在编译期完全确定，不允许 auto 动态推导
    };
    ```
2. **不能直接用于常规函数的参数（C++20 之前）**：在旧标准里，`void func(auto x);` 是不合法的。

3. **无法自动推导数组类型**：
  
    ```C++
    int arr[10] = {0};
    auto a = arr; // ❌ a 会退化（decay）为指针类型 int*，而不是数组类型。
    ```

## 5. 最佳实践：什么时候该用 auto？

虽然 `auto` 不会带来任何运行期性能损失（因为它完全是在编译阶段由编译器把 `auto` 替换为真实类型），但滥用 `auto` 会让代码的可读性变成灾难。

- **强烈建议使用**：
    
    - 类型极其冗长，一眼看不清重点时（如：`std::vector<int>::const_reverse_iterator`）。
        
    - 表达式类型由第三方库决定，未来可能发生变动时（用 `auto` 解耦，防止未来改了底层类型导致全代码跟着改）。
        
    - 接收 Lambda 表达式时。
        
- **不建议使用**：
    
    - 初始化表达式非常模糊，使用 `auto` 让人完全猜不出这是个什么业务对象（例如：`auto result = calculate();`，读者很难一眼看出 `result` 是个整数、浮点数还是一个复杂的自定义结构体）。

> **金句法则**：**如果使用 `auto` 能让代码更专注于逻辑、减少视觉噪音，那就放心去用；如果使用 `auto` 会让读代码的人（包括三个月后的你自己）感到困惑，请老老实实写出类型。**
