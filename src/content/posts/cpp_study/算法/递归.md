## 学会写递归的思维方法

很多人卡住，是因为试图"在脑子里模拟整个递归展开过程"——一层套一层，很快就晕了。**这是错误的学习方法。**

正确的方法是"**信任递归**"，分三步思考：

**第一步：明确函数的语义（最重要！）**  
先用一句话说清楚"这个函数接收什么，返回什么"。比如：`mergeTwoLists(A, B)` 返回 A、B 合并后的有序链表头。**一旦定下语义，就把它当成一个已经写好的、可信的工具。**

**第二步：想终止条件**  
什么情况下问题最简单，简单到不用递归就能直接回答？比如有一条链表空了。

**第三步：想当前层怎么用子问题的结果**  
假设"更小规模的同类问题"已经被正确解决了（这就是信任），我当前这一层只需要做一件小事，把子结果拼装成当前层的答案。比如：选出较小的头，把它的 next 接到"剩余部分的合并结果"上。

**关键心态：你永远只需要想清楚"一层"的逻辑，剩下的交给递归自己。**


### 写递归的五步流程

#### 第一步：定义函数语义（最重要，先想清楚再动手）

**用一句话回答："这个函数接收什么，返回什么？"**

不要去想内部怎么实现，只定义它对外的"契约"。

本题：

> `swapPairs(head)`：接收一个链表头 head，返回这个链表**两两交换之后**的新头节点。

写下来，贴在心里。后面所有思考都基于这个定义。

---

#### 第二步：找终止条件（什么时候不用递归就能直接答）

问自己：**什么情况下问题已经小到不用再拆，可以直接给答案？**

本题：当链表为空，或只剩一个节点时，没有"一对"可以交换，直接返回 head 本身。

cpp

```cpp
if (head == nullptr || head->next == nullptr) {
    return head;
}
```

**小技巧：** 终止条件往往就是"规模最小的输入"。链表题常见的是 `head == nullptr`（空）和 `head->next == nullptr`（单节点）。

---

#### 第三步：确定"当前层"要处理的最小单位

问自己：**这一层我只负责做哪一件小事？剩下的交给递归。**

本题：当前层只负责交换**最前面的这一对**（head 和 head->next），后面的所有对，不归我管，丢给递归。

cpp

```cpp
ListNode* second = head->next;  // 当前这一对是 head 和 second
```

---

#### 第四步：信任递归，拿到子问题的结果

问自己：**假设"处理剩余部分"已经被正确完成了，它会返回什么？我怎么用？**

这是最需要"信任"的一步。本题：剩余部分是从 `second->next` 开始的链表，我信任 `swapPairs(second->next)` 能返回"剩余部分两两交换后的新头"。

cpp

```cpp
ListNode* rest = swapPairs(second->next);  // 信任它已经处理好了
```

---

#### 第五步：用子结果组装当前层的答案

问自己：**拿到子结果后，我当前层怎么拼出完整答案，返回什么？**

本题：交换 head 和 second 这一对——

- head 应该接到"剩余部分的新头"（也就是 rest）
- second 应该指回 head
- 这一对交换后，second 变成了新头，返回 second

cpp

```cpp
head->next = rest;     // head 接到后面处理好的部分
second->next = head;   // second 指回 head，完成交换
return second;         // second 是当前层的新头
```

---

### 把五步拼起来

cpp

```cpp
ListNode* swapPairs(ListNode* head) {
    // 第二步：终止条件
    if (head == nullptr || head->next == nullptr) {
        return head;
    }
    
    // 第三步：当前层处理的最小单位
    ListNode* second = head->next;
    
    // 第四步：信任递归处理剩余部分
    ListNode* rest = swapPairs(second->next);
    
    // 第五步：组装当前层答案
    head->next = rest;
    second->next = head;
    return second;
}
```

---

### 这套流程的通用模板

把这五步抽象成模板，以后套用：

cpp

```cpp
返回类型 递归函数(参数) {
    // ② 终止条件：最小情况直接返回
    if (最小情况) return 直接答案;
    
    // ③ 取出当前层要处理的部分
    当前层数据 = ...;
    
    // ④ 信任递归处理子问题
    子结果 = 递归函数(更小的参数);
    
    // ⑤ 用子结果组装当前层答案并返回
    return 组装(当前层数据, 子结果);
}
```

---

### 给你的思维训练建议

写递归时，**强制自己只关注"一层"，绝对不要在脑子里展开第二层、第三层**。一旦你开始想"那 swapPairs(second->next) 里面又会怎么递归……"，你就会晕。

正确的心态是：写第四步时，把 `swapPairs(second->next)` 当成一个**别人已经写好的、保证正确的函数**，你只管拿它的返回值用。

你可以拿这道题练一遍：闭上眼，按"语义 → 终止 → 当前单位 → 信任递归 → 组装"五步在脑子里过一遍，不看代码自己默写出来。能默写出来，就说明你真的掌握了。
