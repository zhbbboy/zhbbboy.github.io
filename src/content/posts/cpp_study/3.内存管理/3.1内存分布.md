## C/C++内存分布
让我们先来看看下面这段代码：
```cpp
int globalVar = 1;
static int staticGlobalVar = 1;
void Test()
{
	static int staticVar = 1;
	int localVar = 1;
	int num1[10] = { 1, 2, 3, 4 };
	char char2[] = "abcd";
	char* pChar3 = "abcd";
	int* ptr1 = (int*)malloc(sizeof (int)* 4);
	int* ptr2 = (int*)calloc(4, sizeof(int));
	int* ptr3 = (int*)realloc(ptr2, sizeof(int)* 4);
	free(ptr1);
	free(ptr3);
}
```

【说明】
 1、栈又叫堆栈，用于存储非静态局部变量/函数参数/返回值等等，栈是向下增长的。
 2、内存映射段是高效的I/O映射方式，用于装载一个共享的动态内存库。用户可使用系统接口创建共享内存，做进程间通信。
 3、堆用于存储运行时动态内存分配，堆是向上增长的。
 4、数据段又叫静态区，用于存储全局数据和静态数据。
 5、代码段又叫常量区，用于存放可执行的代码和只读常量。

**顺便提一下**：为什么说栈是向下增长的，而堆是向上增长的？
![](../../_Pasted_Image/Pasted%20image%2020260709142156.png)
**简单来说，在一般情况下，在栈区开辟空间，先开辟的空间地址较高，而在堆区开辟空间，先开辟的空间地址较低。**
例如，下面代码中，变量a和变量b存储在栈区，指针c和指针d指向堆区的内存空间：
```cpp
#include <iostream>
using namespace std;
int main()
{
	//栈区开辟空间，先开辟的空间地址高
	int a = 10;
	int b = 20;
	cout << &a << endl;
	cout << &b << endl;

	//堆区开辟空间，先开辟的空间地址低
	int* c = (int*)malloc(sizeof(int)* 10);
	int* d = (int*)malloc(sizeof(int)* 10);
	cout << c << endl;
	cout << d << endl;
	return 0;
}
```
因为在栈区开辟空间，先开辟的空间地址较高，所以打印出来a的地址大于b的地址；在堆区开辟空间，先开辟的空间地址较低，所以c指向的空间地址小于d指向的空间地址。

注意：在堆区开辟空间，后开辟的空间地址不一定比先开辟的空间地址高。因为在堆区，后开辟的空间也有可能位于前面某一被释放的空间位置。

### 从“生命周期”角度理解这些内存区域

你可以按照生命周期来记。
#### 1. 代码区

```
生命周期：程序整个运行期间
内容：函数机器指令
权限：通常只读 + 可执行
```

#### 2. 常量区

```
生命周期：程序整个运行期间
内容：字符串字面量、只读常量
权限：通常只读
```

#### 3. 全局/静态区

```
生命周期：程序整个运行期间
内容：全局变量、static变量
权限：可读写
```

#### 4. 栈区

```
生命周期：跟随函数调用
内容：局部变量、函数参数、返回地址、栈帧信息
特点：自动创建，自动释放，速度快，空间较小
```

#### 5. 堆区

```
生命周期：由程序员控制
内容：malloc/new 动态申请的对象
特点：灵活，空间较大，但需要手动释放或用智能指针管理
```

### 从“为什么分区”角度总结

你可以这样理解：

|区域|为什么要单独存在|
|---|---|
|代码区|指令需要可执行，但不应该被随便修改|
|常量区|常量不应该被修改，可以共享、保护|
|全局/静态区|这些变量需要在程序整个生命周期内存在|
|栈区|函数调用天然符合后进先出，自动分配释放很高效|
|堆区|有些对象大小和生命周期运行时才知道，需要动态管理|

所以不是为了背概念而强行分区，而是因为程序里的东西具有不同特征：

```
有的要执行
有的不能改
有的活到程序结束
有的函数结束就销毁
有的运行时才知道要多大
```

所以内存必须分区域管理。

### 一个非常重要的面试表达

如果面试官问：

> C/C++ 程序内存区域有哪些？

你可以这样答：

```
从常见进程地址空间角度看，C/C++ 程序运行时通常可以分为代码区、只读常量区、全局/静态区、堆区和栈区。

代码区存放函数编译后的机器指令，通常具有只读和可执行权限；常量区存放字符串字面量等只读数据；全局/静态区存放全局变量和 static 变量，生命周期贯穿整个程序运行期间；栈区用于函数调用，保存函数参数、局部变量、返回地址等栈帧信息，由系统自动分配和释放；堆区用于动态内存分配，比如 malloc/new 申请的内存，生命周期由程序员控制，需要 free/delete 或智能指针管理。

这些区域的划分本质上是因为不同数据的生命周期、访问权限、分配时机和管理方式不同。
```

这一段面试说出来就比较完整。

### 最容易混淆的几个点

#### 1. 指针变量在哪里，和它指向哪里，是两回事

```cpp
int* p = new int(10);
```

正确理解：

```
p：局部指针变量，通常在栈上
*p：new 出来的 int 对象，在堆上
```

#### 2. 字符串常量和字符数组不同

```cpp
const char* p = "hello";
```

这里：
```
p 在栈区或全局区，取决于 p 定义在哪里
"hello" 在常量区
```

但是：
```cpp
char arr[] = "hello";
```

如果在函数内：
```cpp
void func()
{
    char arr[] = "hello";
}
```

这里 `arr` 是一个数组，通常在栈区，内容是从字符串字面量拷贝过来的。

区别很大：
```cpp
const char* p = "hello"; // 不能修改 "hello"
char arr[] = "hello";    // 可以修改 arr[0]
```

例如：

```cpp
arr[0] = 'H'; // 可以
p[0] = 'H';   // 错误，未定义行为
```

## 3. static 局部变量不是在栈上

```cpp
void func()
{
    static int count = 0;
    count++;
}
```

虽然 `count` 写在函数里面，但它不是普通局部变量。

它的特点是：

```
作用域：只在 func 函数内可见
生命周期：程序运行期间一直存在
存储位置：全局/静态区
```

所以它不会随着函数结束而销毁。

#### 4. 局部变量不一定真的在栈上

从面试学习角度，你可以说：

```
普通局部变量通常在栈上。
```

但从编译器优化角度，它可能被放到寄存器里，甚至直接优化掉。

例如：

```cpp
void func()
{
    int x = 10;
    cout << x << endl;
}
```

编译器可能根本不为 `x` 分配真实内存，而是直接把 `10` 用到指令里。

所以严谨表达是：

```
局部变量通常具有自动存储期，典型实现中放在栈上，但可能被编译器优化到寄存器或直接消除。
```
