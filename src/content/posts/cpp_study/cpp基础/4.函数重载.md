## 4.1函数重载的概念
核心概念是：**在同一个作用域内，可以声明几个功能类似但“参数列表”不同的同名函数。** 这样做的最大好处是**解放了起名字的痛苦**。例如，无论是计算两个整数相加，还是两个浮点数相加，你都可以用同一个函数名 `add`，而不必苦恼地去起名为 `add_int`、`add_float`。

### 规则：参数的 数量、类型 或 顺序 至少有一个不同

C++

```
// 1. 参数数量不同
void print(int a);
void print(int a, int b);

// 2. 参数类型不同
void print(int a);
void print(double a);

// 3. 参数顺序不同（本质也是类型不同）
void print(int a, double b);
void print(double a, int b);
```

只要满足上述条件之一，它们就是完全不同的函数，编译器可以清晰地分流。
**注意：形参列表不同是指参数个数、参数类型或者参数顺序不同，若仅仅是返回类型不同，则不能构成重载。**
## 4.2 函数重载的底层原理：名字修饰
为什么 C 语言不支持函数重载，而 C++ 可以？这背后的功臣是 C++ 编译器的 **“名字修饰”（Name Mangling）** 技术。

在 C 语言中，编译器生成的符号表里，函数名就是它的汇编标识。如果出现两个 `void add(int);` 和 `void add(float);`，在汇编里它们都叫 `_add`，自然就冲突了。

而 C++ 编译器在编译时，会**暗中把参数列表的信息“揉”进函数名里**，生成一个独一无二的新名字。

举个直观的例子（具体修饰规则因编译器如 GCC、MSVC 而异，这里简化示意）：
- `void add(int a, int b);` $\rightarrow$ 可能会被修饰为 `_Z3addii` （`ii` 代表两个 `int`）
- `void add(double a);` $\rightarrow$ 可能会被修饰为 `_Z3addd` （`d` 代表 `double`）
对于链接器（Linker）来说，这分明就是两个完全不一样的函数，调用时根据你传入的实参类型，精准对应到修饰后的名字即可。因此，**函数重载的开销完全在编译期，运行时是零成本（Zero-cost abstraction）的。**

我们知道，一个C/C++程序要运行起来都需要经历以下几个阶段：预处理、编译、汇编、链接。

我们知道，在编译阶段会将程序中的每个源文件的全局范围的变量符号分别进行汇总。在汇编阶段会给每个源文件汇总出来的符号分配一个地址（若符号只是一个声明，则给其分配一个无意义的地址），然后分别生成一个符号表。最后在链接期间会将每个源文件的符号表进行合并，若不同源文件的符号表中出现了相同的符号，则取合法的地址为合并后的地址（重定位）。

在C语言中，汇编阶段进行符号汇总时，一个函数汇总后的符号就是其函数名，所以当汇总时发现多个相同的函数符号时，编译器便会报错。而C++在进行符号汇总时，对函数的名字修饰做了改动，函数汇总出的符号不再单单是函数的函数名，而是通过其参数的类型和个数以及顺序等信息汇总出 一个符号，这样一来，就算是函数名相同的函数，只要其参数的类型或参数的个数或参数的顺序不同，那么汇总出来的符号也就不同了。

注：不同编译器下，对函数名的修饰不同，但都是一样的。

总结：
 1、C语言不能支持重载，是因为同名函数没办法区分。而C++是通过函数修饰规则来区分的，只要函数的形参列表不同，修饰出来的名字就不一样，也就支持了重载。
 2、另外我们也理解了，为什么函数重载要求参数不同，根返回值没关系。

extern “C”
 有时候在C++工程中可能需要将某些函数按照C的风格来编译，在函数前加“extern C”，意思是告诉编译器，将该函数按照C语言规则来编译。

注意：在函数前加“extern C”后，该函数便不能支持重载了。
