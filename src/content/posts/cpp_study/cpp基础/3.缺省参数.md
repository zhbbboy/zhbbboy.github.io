## 1.缺省参数的概念
**缺省参数（Default Parameters / Default Arguments）**，也常被称为**默认参数**。

它是 C++ 允许在**函数声明或定义**时，为某些参数指定一个“默认值”。如果在调用函数时没有传入对应的实参，编译器就会自动使用这个提前设定好的“缺省值”。

这个特性的核心目的是**简化函数调用**，减少为了处理不同参数组合而重复编写重载函数的麻烦。
```
#include <iostream>
using namespace std;
void Print(int a = 0)
{
	cout << a << endl;
}
int main()
{
	Print();//没有指定实参，使用参数的默认值（打印0）
	Print(10);//指定了实参，使用指定的实参（打印10）
	return 0;
}

```
## 2.缺省函数的分类
分为全缺省参数，半缺省参数

### 2.1 全缺省参数
全缺省参数，即函数的全部形参都设置为默认参数/缺省参数
```
void Print(int a =0,int b =20 ,int c =30)
{
	count << a << endl;
	count << b << endl;
	count << c << endl;
}
```

半缺省参数
半缺省参数，即函数的参数不全为缺省参数。
```
void Print(int a, int b, int c = 30) { 
	cout << a << endl; 
	cout << b << endl; 
	cout << c << endl; 
}
```
注意：
1. 半缺省参数必须从右往左依次给出，不能间隔着给。
2. 缺省参数不能在函数声明和定义中同时出现
```
//错误示例
//test.h
void Print(int a, int b, int c = 30);
//test.c
void Print(int a, int b, int c = 30)
{
	cout << a << endl;
	cout << b << endl;
	cout << c << endl;
}
```
缺省参数只能在函数声明时出现，或者函数定义时出现（二者之一均正确）。
3. 缺省值必须是常量或者全局变量。
```
//正确示例
int x = 30;//全局变量
void Print(int a, int b = 20, int c = x)
{
	cout << a << endl;
	cout << b << endl;
	cout << c << endl;
}

```

## 3. 缺省参数的底层原理

缺省参数并不是什么魔法，它其实是**编译器帮你做了“脏活累活”**。
```
void Print(int a,int b =20 ,int c =30)
{
	count << a << endl;
	count << b << endl;
	count << c << endl;
}
```

在编译阶段，当编译器发现你调用 `Print(5)` 时，它会去查看该函数的声明，发现后面两个参数有默认值。于是，编译器在生成的汇编代码中，**自动帮你把 `20` 和 `30` 压入栈中（或者放入寄存器）**。

也就是说，在最终生成的机器码中： `Print(5.0);` **等价于** `Print(5,20,30)`

两者在运行时的性能是**完全没有任何区别**的。

## 4. 注意事项与潜在的坑

### 1. 与函数重载（Overload）的冲突（二义性）

缺省参数和函数重载一起使用时，非常容易引发编译器“选择困难症”：

C++

```
void printInfo(string name) { ... }
void printInfo(string name, int age = 18) { ... } // 带有缺省参数

// 调用时：
printInfo("Tom"); // 编译报错！产生二义性（Ambiguous call）
```

此时编译器不知道该去调用第一个单参数的函数，还是调用第二个带有默认参数的函数，只能罢工报错。

### 2. 缺省值可以是动态的表达式

C++ 的缺省值不一定非得是硬编码的字面量（如 `10`、`"Red"`），它也可以是全局变量、全局函数甚至表达式（只要在编译或调用时能确定即可）。不过为了代码可读性，通常建议使用常量。
